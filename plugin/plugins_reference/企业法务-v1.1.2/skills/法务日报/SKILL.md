# /法务日报 -- 法务工作简报生成

> 如遇到不熟悉的占位符或需检查已连接工具，参见 [CONNECTORS.md](../../CONNECTORS.md)。

聚合法务团队的工作信息，生成结构化法务日报或周报。用户手动输入工作内容，系统自动分类整理并生成专业简报。

**重要提示**：本技能辅助法务团队日常工作信息整理，不替代专业法律分析。生成的简报中涉及的风险提示仅供参考。

**独立能力（无需连接器）**

- 用户手动输入法务工作内容，生成结构化日报/周报
- 智能分类整理（合同/合规/诉讼/到期提醒/监管动态）
- 风险预警标注（紧急事项自动高亮）
- 输出日报和周报两种模式
- 下周重点工作规划建议

**增强能力（连接器加持）**

- ~~Notion -> 将法务日报/周报写入Notion知识库，便于团队查阅和知识沉淀

## 连接器（可选增强）

| 连接器 | 增强能力 |

| **~~Notion** | 将法务日报/周报写入Notion知识库页面，便于团队查阅和历史追溯 |

## 调用方式

```
/法务日报 日报
/法务日报 周报
/法务日报 <直接描述本周工作内容>
```

生成法务工作简报：@$1

## 输入要求

用户提供本周法务工作内容，可以是：
1. **工作清单**：逐条列出本周处理的法务事项
2. **自由描述**：自然语言描述本周工作情况
3. **混合输入**：提供部分信息，系统引导补充

接受材料后，确认以下信息：
1. **报告类型**：日报 / 周报
2. **时间范围**（可选）：默认日报为当天，周报为本周一至今
3. **关注重点**（可选）：如"重点关注合同到期情况"

如用户未提供充分信息，引导用户按类别补充：
1. 本周处理了哪些合同？
2. 合规事项有什么进展？
3. 有没有诉讼/仲裁的最新进展？
4. 有什么即将到期的事项？
5. 收到了什么监管通知或外部律师反馈？

## 执行流程

### 第一步：智能分类和摘要

将采集到的信息按以下类别整理：

| 类别 | 包含内容 | 排序规则 |
|------|---------|---------|
| **合同审查进展** | 新增合同审查、审查中的合同状态、已完成的合同签署 | 按紧急程度排序 |
| **合规事项** | 合规整改进展、新发现的合规风险、监管动态 | 按风险等级排序 |
| **诉讼/仲裁进展** | 案件最新状态、庭审安排、律师沟通摘要 | 按开庭日期排序 |
| **到期提醒** | 合同到期、整改截止、答复期限、申诉时效 | 按到期日排序（最近优先） |
| **监管动态** | 新政策发布、监管通知、行业合规要求变化 | 按发布时间排序 |
| **内部需求** | 来自业务部门的法务咨询/审核请求 | 按提出时间排序 |

对每条信息生成简洁摘要（一句话概括+当前状态+需要的下一步行动）。

### 第二步：风险预警

自动标注需要立即处理的紧急事项，触发条件：

| 预警级别 | 触发条件 | 标注方式 |
|---------|---------|---------|
| **红色（立即处理）** | 合同明天到期未签署；整改截止日在3天内；庭审在本周且未准备材料；监管要求当日响应 | [紧急] 前缀标注 |
| **橙色（本周处理）** | 合同本周到期；待办已逾期；律师等待回复超过3天 | [重要] 前缀标注 |
| **黄色（关注）** | 合同2周内到期；合规整改即将到截止日；潜在风险提前预警 | [关注] 前缀标注 |

### 第三步：生成报告

根据报告类型输出对应格式：

**日报模式（聚焦当日紧急事项）：**

```
## 法务日报 -- [日期]

### 紧急事项（需今日处理）
[红色预警事项，附具体行动建议]

### 今日待办
[今日应完成的法务工作事项]

### 今日动态
- **合同**：[今日合同相关进展]
- **诉讼**：[今日诉讼相关进展]
- **合规**：[今日合规相关动态]

### 明日预告
[明天需要关注的事项]

### 数据来源
[标注信息来源：手动输入]
```

**周报模式（完整总结+下周计划）：**

```
## 法务周报 -- [起始日期] 至 [结束日期]

### 本周总览
- 合同审查：完成[X]份，进行中[X]份，新增[X]份
- 合规事项：完成整改[X]项，新增[X]项
- 诉讼/仲裁：有进展[X]件
- 内部咨询：处理[X]件

### 风险预警
[按紧急程度排列的风险事项]

### 分类详情

#### 一、合同审查进展
| 合同名称 | 对方主体 | 状态 | 本周进展 | 下一步 |
|---------|---------|------|---------|--------|

#### 二、合规事项
| 事项 | 进展 | 整改截止日 | 风险等级 |
|------|------|-----------|---------|

#### 三、诉讼/仲裁进展
| 案件名称 | 案号 | 本周进展 | 下次节点 |
|---------|------|---------|---------|

#### 四、到期提醒（未来2周）
| 事项 | 到期日 | 剩余天数 | 需要的行动 |
|------|--------|---------|-----------|

#### 五、监管动态
[本周新发布的与企业相关的法规/政策/通知]

### 下周重点
1. [需优先处理的事项]
2. [即将到期的事项]
3. [需要准备的工作]

### 数据来源与局限
[标注信息来源及可能的遗漏]
```

### 第四步：报告归档

**如已连接 ~~Notion：**
- 将法务日报/周报写入Notion知识库页面
- 按日期归档，便于历史查阅和团队协作

## 质量标准

- 每个事项必须包含：摘要+当前状态+需要的行动（不能只列清单不给行动建议）
- 到期提醒必须精确到日期，并计算剩余天数
- 风险预警必须有明确的触发条件和处理建议
- 周报中的数据统计必须准确可验证
- 标注信息来源（手动输入），让读者知道信息的可靠性
- 如有信息缺口，必须在报告中明确标注
- 日报控制在1页以内（聚焦紧急事项），周报不超过3页
- 摘要语言简洁专业，避免冗余描述

## 使用建议

- 建议配合定时任务，每日早9点提醒生成日报
- 周五下午生成周报，便于团队复盘
- 如有红色预警事项，建议立即处理

## 关联技能

- **合规审查** -- 对周报中发现的合规风险做深入审查
- **风险评估** -- 对紧急事项做系统性风险评估
- **企业背景调查** -- 对新合同对方主体做背景调查
- **法律文书** -- 根据到期提醒起草催告函或续约通知
