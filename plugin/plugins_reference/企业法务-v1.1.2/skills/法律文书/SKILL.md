# /法律文书 -- 法律文书自动适配与生成

> 如遇到不熟悉的占位符或需检查已连接工具，参见 [CONNECTORS.md](../../CONNECTORS.md)。

根据事实背景和诉求自动识别文书类型，按照中国裁判文书格式规范和律师行业标准生成法律文书初稿。

**重要提示**：本技能辅助法律文书起草工作流程，不构成法律意见。生成的文书为初稿，需执业律师审核后方可使用。

**独立能力（无需连接器）**

- 自动识别文书类型（7种）
- 按规范格式生成完整文书初稿
- 法律依据和证据清单
- 诉讼时效和管辖权提示
- 文书风格与语气适配
- 高频起草错误自动排查

**增强能力（连接器加持）**

- ~~Notion -> 将法律文书写入Notion知识库，便于律师团队协作审阅修改

## 连接器（可选增强）

| 连接器 | 增强能力 |
|-----------|---------|
| **~~Notion** | 将法律文书写入Notion知识库页面，便于律师团队协作审阅修改 |

> 没有连接器也完全可以使用——基于内置知识引用核心法条（标注"[建议核实原文]"）。

## 调用方式

```
/法律文书 <事实和诉求描述>
```

根据以下情况起草法律文书：@$1

接受事实描述（文本）、相关材料（合同/通知/证据等文件）或直接指定文书类型。

## 输入要求

1. **文书类型**：律师函/法律意见书/起诉状/答辩状/代理词/仲裁申请书/执行申请书——如未指定则根据事实自动推荐
2. **当事人信息**：发函方/原告/被告的名称、地址、联系方式；法人须含统一社会信用代码
3. **事实背景**：争议起因、合同关系、违约行为、损失情况
4. **诉求/目的**：希望达成的法律效果
5. **证据情况**（如适用）：已有证据清单
6. **特殊要求**（可选）：语气偏好、是否需要律所抬头、是否已确定管辖法院

用户只描述大致情况也可继续，对缺失信息做合理假设并标注。

## 执行流程

### 第一步：识别文书类型

| 场景特征 | 推荐文书类型 |
|---------|------------|
| 催告履行/警告停止/通知解除 | 律师函 |
| 交易风险分析/合规论证/决策支持 | 法律意见书 |
| 主动起诉/索赔 | 民事起诉状 |
| 收到起诉状后应诉 | 民事答辩状 |
| 庭审发言 | 代理词 |
| 劳动争议/合同约定仲裁 | 仲裁申请书 |
| 已有生效判决需执行 | 执行申请书 |

### 文书语气与风格指引

| 文书类型 | 语气基调 | 风格要求 | 忌讳 |
|---------|---------|---------|------|
| **律师函** | 严正但专业 | 事实清晰+法律后果明确+回复期限具体 | 情绪化措辞、人身攻击、威胁恐吓 |
| **法律意见书** | 客观分析 | 论证逻辑严密+结论附保留条件+风险提示充分 | 武断结论、忽略不利因素、遗漏假设 |
| **起诉状** | 严谨精确 | 事实简洁客观+请求明确可执行+法条逐一对应 | 感情色彩、无关事实堆砌、请求模糊 |
| **答辩状** | 有理有据 | 逐条回应诉请+提出反证+指出原告逻辑漏洞 | 回避核心争议、笼统否认、情绪化反驳 |
| **代理词** | 说服力强 | 归纳争议焦点+分层论证+引用判例支撑 | 重复起诉状/答辩状内容、论点散乱 |
| **仲裁申请书** | 严谨精确 | 同起诉状，注意仲裁条款/协议引用 | 遗漏仲裁管辖依据 |

**文书格式要素**（按类型自动加载）：

**律师函**：律所抬头+编号+致函对象+受委托说明+事实陈述+法律依据+律师意见和要求+回复期限+签名盖章+日期。

**法律意见书**：律所抬头+编号+引言（委托背景+审查范围+假设与限制）+事实概要+法律分析（逐项论证）+意见结论+保留声明+签名。

**民事起诉状**：标题+原告信息（自然人六要素/法人四要素）+被告信息+诉讼请求+事实和理由+证据清单+管辖法院+签章+日期。

**民事答辩状**：标题+答辩人信息+被答辩人信息+答辩请求+逐条回应原告诉请+证据清单+签章+日期。

**代理词**：标题+案由和当事人信息+代理意见（事实认定+法律适用+证据分析+反驳对方观点）+结论+签名+日期。

### 第二步：组织事实和法律依据

**事实部分**：按时间线梳理，区分无争议/有争议事实，标明关键合同条款和违约行为。

**法律依据部分**：

基于内置知识引用核心法条，标注"[建议核实原文]"。

法条引用格式：《XXX》第X条第X款第X项。引用优先级：法律 > 行政法规 > 司法解释 > 地方性法规。

**证据部分**（起诉状/答辩状）：编号排列，每项标注证据名称、证据类型、证明目的。

### 第三步：按规范格式生成文书

格式规范：
- 当事人信息完整（自然人：姓名+性别+民族+出生日期+身份证号+住址+联系方式；法人：名称+住所+法定代表人+统一社会信用代码）
- 金额大写规范（壹贰叁肆伍陆柒捌玖拾佰仟万亿）
- 日期格式：XXXX年XX月XX日
- 法条逐一列明，不笼统引用
- 诉讼请求/律师要求须具体、明确、可执行

### 第四步：最终检查

| 检查项 | 说明 |
|--------|------|
| 诉讼时效 | 普通时效3年（民法典第188条），特殊时效1年（人身损害等） |
| 管辖权 | 约定管辖优先，否则被告住所地/合同履行地 |
| 诉讼费 | 是否需要预估，是否请求被告承担 |
| 保全请求 | 是否需要财产保全/证据保全/行为保全 |
| 先予执行 | 追索赡养费/劳动报酬等是否可申请先予执行 |
| 律师函回复期 | 通常7-15个工作日 |

### 高频起草错误清单

按文书类型列出常见错误，生成后须逐项排查：

| 文书类型 | 常见错误 | 正确做法 |
|---------|---------|---------|
| **律师函** | 未写明回复期限和后果 | 明确"请于收函后X个工作日内..."，逾期将采取法律行动 |
| **律师函** | 未附委托授权说明 | 首段写明"受XX委托，就XX事宜致函如下" |
| **起诉状** | 诉讼请求表述不可执行（如"赔偿合理损失"） | 具体到金额和计算方式（如"赔偿XX元及利息"） |
| **起诉状** | 遗漏诉讼费承担请求 | 诉请最后一项加"本案诉讼费用由被告承担" |
| **答辩状** | 笼统否认全部诉请 | 逐条回应，承认无争议事实，集中反驳争议点 |
| **代理词** | 照搬起诉状/答辩状内容 | 基于庭审情况归纳争议焦点，针对性论证 |
| **法律意见书** | 遗漏假设条件和免责声明 | 引言写明"本意见基于以下假设..."，结尾附保留声明 |
| **仲裁申请书** | 遗漏仲裁条款引用 | 明确引用仲裁协议/合同仲裁条款全文 |

## 质量标准

- 当事人信息格式必须完整符合规范
- 法条引用必须精确到条款项，不得笼统引用"相关法律法规"
- 金额表述必须包含大小写
- 诉讼请求必须具体可执行
- 管辖法院必须给出判断依据
- 诉讼时效必须主动提示
- 不得使用模板化的空泛措辞
- 文书语气须与类型匹配（参照风格指引矩阵）
- 生成后须自动排查高频起草错误

## 关联技能

- **查案例** -- 先做类案检索，再据此起草更有针对性的文书
- **风险评估** -- 评估诉讼事项的整体法律风险
- **公司决议** -- 如需配套公司内部决议（如授权诉讼的决议）
