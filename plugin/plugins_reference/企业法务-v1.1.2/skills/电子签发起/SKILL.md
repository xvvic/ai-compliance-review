# /电子签发起 -- 合同电子签署流程

> 如遇到不熟悉的占位符或需检查已连接工具，参见 [CONNECTORS.md](../../CONNECTORS.md)。

准备合同文件并规划电子签署流程——执行签前检查、配置签署参数、生成签署操作指引和邮件模板、规划签后处理。

**重要提示**：发起签署前请确认合同已完成内部审批流程。本技能辅助签署流程管理，不替代合同审查。

**独立能力（无需连接器）**

- 签前合规检查清单（7项核验）
- 签署配置方案生成（顺序/并行/审批流）
- 签署路由建议（基于合同类型和金额）
- 签署操作指引和邮件模板
- 催签话术模板
- 签后处理提醒清单

**增强能力（连接器加持）**

- ~~Notion -> 将签署配置、状态跟踪和签后处理记录写入Notion知识库

## 连接器（可选增强）

## 调用方式

```
/电子签发起 <合同信息描述>
```

准备发起签署：@$1

## 输入要求

接受以下信息（可逐步引导补充）：

1. **合同文件**：文件名/链接/描述（如"与XX公司的SaaS服务协议"）
2. **合同类型**：NDA / 服务协议 / 采购合同 / 劳动合同 / 框架协议 / 补充协议 / 其他
3. **签署方信息**：各方签署人姓名、邮箱、职务
4. **签署顺序**（可选）：并行签署 / 串行签署（先内后外） / 需内部审批后再发送
5. **合同金额**（可选）：影响审批流程和签署路由
6. **截止日期**（可选）：期望完成签署的日期
7. **签署模式**（可选）：单份签署 / 批量签署（多份同类合同）

用户只提供合同基本信息也可继续，对缺失信息做合理提示。

## 执行流程

### 第一步：签前检查清单

在发起签署前，逐项验证以下要点：

| 检查项 | 验证内容 | 通过标准 |
|--------|---------|---------|
| 合同终稿确认 | 合同是否为最终版本，无待确认的修改 | 无红线标注/修订痕迹/待确认条款 |
| 内部审批完成 | 是否已通过内部审批流程 | 法务审核通过+业务负责人确认+（如适用）财务确认 |
| 签署方信息完整 | 各方签署人的姓名、邮箱、职务是否准确 | 签署人经对方确认+有签署授权 |
| 法律主体正确 | 签署页上的公司名称是否为正确的法律实体 | 与营业执照/公司注册信息一致 |
| 附件齐全 | 所有附件、附录、补充协议是否完整附上 | 合同正文引用的附件全部包含 |
| 签署授权确认 | 签署人是否有权代表公司签署该合同 | 有法定代表人签字/公司决议授权/授权委托书 |
| 日期处理 | 合同日期是留空（签署时自动填充）还是已填写 | 明确日期处理方式 |

**检查结果判定**：
- 全部通过：可以发起签署
- 存在未通过项：列出问题清单，暂停发起直至解决

**常见签前错误防范**：

| 错误类型 | 后果 | 防范措施 |
|---------|------|---------|
| 法律实体名称拼写错误 | 合同效力争议 | 与工商登记信息逐字核对 |
| 签署人无授权 | 合同可被撤销/无效 | 确认授权委托书或法定代表人身份 |
| 附件遗漏 | 合同不完整，履约争议 | 对照正文引用逐一核对附件 |
| 版本混乱（非终稿） | 需重新签署，延误项目 | 确认文件名含"FINAL"或最终审批记录 |

### 第二步：签署配置

根据合同类型和企业实际情况配置签署参数：

**签署模式选择**：

| 模式 | 适用场景 | 配置说明 |
|------|---------|---------|
| **并行签署** | 双方地位对等的协议（NDA、框架协议） | 各方同时收到签署请求，任意顺序完成 |
| **串行签署（先内后外）** | 大多数合同 | 内部签署人先签 -> 对方签署人后签 |
| **串行签署（先外后内）** | 需要对方先承诺的场景 | 对方先签 -> 内部再签（如录用通知） |
| **审批后签署** | 需要内部审批的高金额合同 | 内部审批人先审批通过 -> 再发送给签署人 |

**认证方式配置**：

| 认证方式 | 安全等级 | 适用场景 |
|---------|---------|---------|
| 邮件验证 | 基础 | 内部文件、低风险协议 |
| 短信验证码 | 中等 | 标准商业合同 |
| 身份证验证 | 高 | 高金额合同、涉及个人权益的协议 |
| 人脸识别 | 最高 | 重大交易、监管要求 |

**提醒设置建议**：
- 签署截止日期：根据合同紧急程度设置（常规7天，紧急3天）
- 自动提醒：签署请求发出后第3天、第5天提醒未签署方
- 过期处理：到期未签署时通知发起人

### 第三步：签署路由

根据合同类型和金额，建议签署审批流程：

| 合同类型/金额 | 建议签署流程 | 说明 |
|-------------|------------|------|
| NDA / 标准保密协议 | 法务确认 -> 直接签署 | 模板化协议，风险可控 |
| 服务协议 <=50万 | 法务审核 -> 业务负责人签署 | 常规业务合同 |
| 服务协议 50-100万 | 法务审核 -> 部门总监审批 -> 签署 | 中等金额需加审批 |
| 合同金额 >100万 | 法务审核 -> 部门总监 -> 法务总监/CFO审批 -> 签署 | 重大合同须高管审批 |
| 涉及知识产权/排他性 | 法务审核 -> 法务总监审批 -> 签署 | 特殊条款需法务总监把关 |
| 关联交易合同 | 法务审核 -> 董事会决议 -> 签署 | 按公司法要求需要公司决议 |

**自定义路由**：如企业有自己的审批制度，按企业制度优先。上表仅作为无明确制度时的建议参考。

### 第四步：生成签署操作指引

生成完整的签署配置清单和操作指引：

1. **签署配置清单**：含所有配置信息的结构化文档
2. **签署平台操作指引**：
   - 登录签署平台 -> 新建签署任务 -> 上传文档
   - 添加签署人（逐个录入姓名+邮箱）
   - 配置签署字段到正确位置（签名、日期、职务、公司名称）
   - 设置签署顺序和认证方式
   - 设置过期日期和提醒规则
   - 预览并发送
3. **签署邀请邮件模板**（可直接复制发送）
4. **催签话术模板**

**签署字段配置建议**：

为每个签署人配置以下字段：
- **签名字段**：放置在签署页"签字/盖章"处
- **日期字段**：签名旁自动填充签署日期
- **姓名字段**：签署人姓名确认
- **职务字段**：签署人职务
- **公司名称字段**：法律实体名称

**批量签署支持**：

当需要同时发起多份同类合同签署时（如批量劳动合同续签、多个供应商NDA）：

1. 确认合同模板一致性
2. 准备签署人清单（姓名+邮箱的列表）
3. 生成批量签署跟踪表：

| 序号 | 合同名称 | 签署人 | 发送状态 | 签署状态 |
|------|---------|--------|---------|---------|

### 第五步：签署状态跟踪指引

签署发出后的状态跟踪：

| 状态 | 含义 | 建议动作 |
|------|------|---------|
| 已发送 | 签署请求已发出，等待签署 | 等待，无需操作 |
| 已查看 | 签署人已查看 | 等待签署 |
| 已签署 | 全部签署人完成签署 | 触发签后处理 |
| 已拒绝 | 签署人拒绝签署 | 立即通知法务经办人，了解拒签原因 |
| 已过期 | 超过截止日期未完成 | 联系未签署方，评估是否重新发送 |

**异常处理流程**：
- 拒签 -> 通知法务 -> 了解原因 -> 协商修改 -> 重新发起
- 过期 -> 电话/微信催促 -> 延长期限重发 或 升级处理
- 邮箱错误（退信）-> 联系对方确认正确邮箱 -> 更正后重发

### 第六步：签后处理

签署完成后的标准动作清单：

| 处理项 | 动作 | 时限 |
|--------|------|------|
| 合同归档 | 将已签合同存档至合同管理系统/文件夹 | 签署完成当天 |
| 台账更新 | 更新合同台账（状态->已签署、签署日期、到期日） | 签署完成当天 |
| 通知相关方 | 通知业务负责人、财务（如涉及付款）、对接人 | 签署完成当天 |
| 到期提醒设置 | 设置合同到期前30天/60天/90天提醒 | 签署完成一周内 |
| 关键日期记录 | 记录付款日、交付日、续约决定日等关键节点 | 签署完成一周内 |

**如已连接 ~~Notion：**
- 将签署配置方案和签后处理记录写入Notion知识库页面
- 便于团队跟踪签署进度和合同管理

## 质量标准

- 签前检查必须逐项执行，任一关键项未通过不得建议发起签署
- 签署人信息（尤其是法律实体名称）必须准确核对
- 签署路由建议必须考虑合同金额和类型，不能一刀切
- 配置清单必须具体到操作级别（不说"配置好签署顺序"，要说"第一顺位：张三，邮箱xx，角色：甲方法定代表人"）
- 生成的操作指引和邮件模板必须可以直接使用
- 签后处理不可遗漏（归档+台账+通知+提醒四项缺一不可）
- 签署字段配置必须考虑中文合同的签署页布局习惯
- 批量签署须生成完整跟踪表，不遗漏任何一份

## 关联技能

- **合规审查** -- 签署前确认合同内容的合规性
- **风险评估** -- 评估合同签署的法律风险
- **公司决议** -- 如合同签署需要公司决议授权
- **法务日报** -- 跟踪签署进展并在日报/周报中体现
