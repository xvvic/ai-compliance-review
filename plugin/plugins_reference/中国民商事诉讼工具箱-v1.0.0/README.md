# 中国民商事诉讼工具箱

> Qoder for CN Litigation — 面向执业律师与公司法务的全流程诉讼工具箱

## 概览

本插件为中国民商事诉讼提供从案件分析到文书生成的完整工作流支持，覆盖主诉与被诉双视角。基于请求权基础方法和法庭报告技术，将方法论转化为实务落地工具。

**目标用户**：执业律师、公司法务

**技能总数**：20项

## 工作流关联

```
文档解析 ── 上传文件（PDF/图片/Word/PPT）自动解析为 Markdown 后进入分析/起草（用户可配置解析后端，内置 pypdf/Read 降级）

案件事实梳理 ──产出──→ 事实梳理主文档(md) ⇒ 链接 关系图(HTML) + 时间轴图(HTML) + 内含大事记
     │
     ↓
要件攻防分析 ──依赖──→ 案件事实梳理
     │
     ↓
┌────┼────┬────────┐
│    │    │        │
主诉诉状  被诉答辩状  证据目录  质证意见

└────┬────┘
     ↓
  庭审提纲（含询问清单）
     │
     ↓（庭前/庭后）
代理词（庭前代理意见 + 庭后补充代理意见，覆盖一/二/再审）

程序性文书系列 ── 八大家族路由（保全/证据/当事人与诉请变动/庭审程序/执行/立案/其他程序/管辖权异议），含管辖权异议
财产线索调查 ── 独立（可衔接 程序性文书系列 的保全/网络查控）
法律研究报告 ── 独立（研究结论可输入要件攻防分析/主诉诉状/被诉答辩状）
新法解读 ── 独立（新旧对比驱动的律所级新规解读，可衔接法律研究报告/要件攻防分析）

法律核验 ── 质控围栏：以上任一会产出正式文件的技能，在交付 md 成品（及后续转 Word）前，先经法律核验逐条核验法规/法条/案例引用（输出前必过）；放行后默认交付 md，再询问是否转 Word

— 案件管理板块（贯穿案件全生命周期，律师/法务双模式，COLLAB.*协作平台+qmind）—
案件管家 ──建档（律师过冲突门禁/法务做委外评估）──→ 期限计算（算首批期限·建四档待办·可选定时提醒）
                 │
收判决/结案 ──→ 案件复盘（庭审复盘/裁判文书研析/结案复盘）──→ 案件管家（结案+归档）
                                                              │
                                                              ↓
诉讼知识库 ──沉淀──→ 法律研究报告/新法解读/案件复盘成果汇入 qmind/钉钉文档（可被后续检索复用）
```

## 安装

将本目录放置到 QoderWork 可访问的位置，然后在 QoderWork 中安装插件：

```
设置 → 插件管理 → 安装本地插件 → 选择本目录
```

或将目录复制到 `~/.qoderwork/plugins-custom/qoder-for-cn-litigation/`。

---

## 首次使用

安装后首次使用，建议运行 `/冷启动访谈` 完成实践画像配置。套件会根据你的角色（法务/律师）、立场偏好（主诉/被诉）、风险校准等信息，为后续所有技能提供个性化调整。

如果跳过冷启动，所有技能仍可正常使用，但不会有个性化推荐。

## MCP 依赖（可选增强 · 平台无关 · 多供应商灾备）

本插件的所有 MCP 依赖均为**可选**——不连接任何 MCP 也能产出完整草稿。法律检索/工商信息/文档解析/协作平台四类后端均**平台无关**：各技能只按能力槽（`LAW.*`/`BIZ.*`/`PARSE.*`/`COLLAB.*`）调用，运行时用 `qw_mcp_list` 探测已连接 MCP 并映射，同类配多家时**自动灾备**（一家不可用立即切换）。能力↔供应商映射见 `qoder.md`「外部能力后端」与 `connectors.md`。连接后获得增强能力：

| 能力后端 | 能力槽 | 可选供应商（可插拔/灾备） | 缺位降级 |
|---------|---------|---------|---------|
| 法律检索后端 | `LAW.*`（法条/法规/案例检索·详情·语义·引用核验） | 元典 / 北大法宝（可单用、可都连做灾备） | 标注 `[L4-法条待验证]` |
| 工商信息后端 | `BIZ.*`（主体核验·股权·投资·风险·知产·关联） | 企查查 / 天眼查 / 启信宝等同类（按实际功能自适配） | WebSearch 降级，标注 `[L4-主体信息待验证]` |
| 可视化 | — | 无（内置 HTML/SVG 输出，零外部依赖） | — |
| 全网搜索（SEARCH.*） | `SEARCH.web_search` / `SEARCH.ai_search` | 运行时探测（GoogleWebSearch / IflowWebSearch / IflowAiSearch / Tavily 等） | WebFetch 降级 |
| 文档解析后端 | `PARSE.to_markdown` | 用户自选后端（MinerU / 阿里云百炼文档转Markdown / markdownify / 自部署同类，多后端可选+灾备） | 见下方降级链 |
| 协作平台后端 | `COLLAB.*`（表格/文档/通讯录/日历/邮箱） | 钉钉（经 dws）/ 飞书（经 lark-cli）/ 同类 OA；选其一即可 | 本地 JSON+Markdown 降级 |

**配置方式**：在 QoderWork 的「连接器」设置中启用对应的 MCP 服务即可，无需修改插件文件。套件运行时自动探测已连接的后端并使用，同类配多家则自动灾备。

### 案件管理（律师/法务双模式）

| 技能 | 说明 |
|------|------|
| `/案件管家` | 案件全生命周期管理（建档/跟进/期限/全景/结案/归档），律师/法务双模式——律师含利益冲突门禁（《律师法》§39）与委托合同管理，法务含外部律师管理。最小必要字段集适配所有法务和律师场景；含民诉法各类期间+执行保全专项期限计算、四档待办与可选定时提醒；协作平台结构化表格（`COLLAB.sheet_*`，钉钉/飞书/同类）优先、本地文件降级 |
| `/案件复盘` | 三级覆盖：庭审复盘（五维）/裁判文书研析（六维）/结案复盘报告（十项+四维反思）；自办（调取台账+事实+过程文书深度复盘）与非自办（仅基于受限信息）双深度；法务模式附外部律师评价；复盘报告输出前必过 `/法律核验` |
| `/诉讼知识库` | 办案笔记/工作笔记/法律知识库三板块，基于 qmind 语义检索为主 + 协作平台文档（`COLLAB.doc_*`）为团队协作备选；冷启动引导配置 |

### 质控围栏 · 输出侧闸门

| 技能 | 说明 |
|------|------|
| `/法律核验` | **每次输出文件前的必过闸门**：经法律检索后端（元典/北大法宝可插拔+灾备）逐条核验法规/法条/案例引用，判定"准确/时效风险/语义不一致/未命中"，放行才落地；亦可独立核验任意文本 |

## 设计原则

1. **轻量骨架**：提供方法论内核与文书能力框架，不过度预设，留白让用户生长
2. **优雅降级**：每个 MCP 依赖都有降级路径，不连接任何 MCP 也能产出草稿
3. **用户可配置**：通过冷启动访谈 + qoder.md 个性化；后续通过 `/自定义` 单项快改
4. **平台无关**：不绑定特定 MCP 供应商，运行时自动探测使用
5. **方法论自主**：核心分析引擎内置方法论，不捆绑外部资源
6. **可溯源**：四级溯源标签体系，引用可靠度透明

---

## 方法论基础

本插件的核心分析引擎（要件攻防分析）采用以下方法论框架：

- **请求权基础方法**：从诉请法律效果反向检索规范基础，逐要件审查
- **法庭报告技术**：原告阶段→被告阶段→反抗辩→再抗辩→证据阶段的递进审查
- **三层审查结构**：权利已产生→权利未消灭→权利可行使，逐层递进、任一层否定则整体否定
