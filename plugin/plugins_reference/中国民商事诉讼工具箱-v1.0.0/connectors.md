# 连接器依赖说明 (CONNECTORS)

> 本套件设计为**无连接器即可完整使用**。连接器为可选增强，接入后能力更强、自动化程度更高。

---

## 设计原则

1. **无连接器可用**：所有技能在无任何 MCP 连接时仍可正常工作，仅能力降级
2. **优雅降级**：每个连接器不可用时，技能自动切换降级方案并标注
3. **按需连接**：用户可根据实际权限逐步接入连接器
4. **平台无关 + 多供应商灾备**：法律检索、工商信息、文档解析、协作平台四类外部能力均**面向"能力"而非某一家工具名**（`LAW.*` / `BIZ.*` / `PARSE.*` / `COLLAB.*`）。同类可配多家供应商，用户用哪家都能跑；配多家则自动灾备——首选不可用立即切备用。skill 内部只声明所需能力，运行时经 `qw_mcp_list` 探测已连接 MCP 并映射。详见 `qoder.md`。
5. **输出 md 优先**：所有文书技能默认先产出 Markdown 交付物，经 `/法律核验` 放行后再询问"是否转 Word（Word 转换后端（DOCX.md_to_docx））"；连接器仅作可选增强，不改变 md 优先的输出底座。

## 连接器一览

### 法律检索后端（元典 / 北大法宝，可单用·可灾备）

> **平台无关**：法律检索是一类**能力**，由元典（华宇元典开放平台）或北大法宝（pkulaw）等任一供应商提供。用户配其一即可用，配两家则自动灾备。技能内部只用能力名（`LAW.*`），不绑定供应商工具名。

| 项目 | 说明 |
|------|------|
| **用途** | 法条精确检索、语义检索、法规详情查询、案例检索、现行有效性验证、引用一键核验（幻觉校验） |
| **供应商** | 华宇元典（默认）/ 北大法宝 / 其他同类法律数据库 MCP；可单选或都配，都配时按首选→灾备 |
| **使用技能** | 法律核验（输出前必过闸门）/ 要件攻防分析 / 主诉诉状 / 被诉答辩状 / 质证意见 / 庭审提纲 / 代理词 / 程序性文书系列 / 证据目录 / 法律研究报告 / 新法解读 |
| **核心价值** | 确保引用法条现行有效，消除 [L4-法条待验证] 标记；获取类案裁判观点支撑诉讼策略 |
| **降级方案** | 全部供应商不可用时：法条标 [L4-法条待验证]，案例引用标 [L4-案例待验证]，建议用户通过裁判文书网/官方法规库人工检索。配置路径：QoderWork → 设置 → 连接器 → 搜索"元典"或"北大法宝" |

**抽象能力槽**（skill 统一用左列名，运行时映射到实际工具）：

| 能力槽 | 含义 | 元典官方 | 自部署/同类 | 北大法宝(运行时发现) |
|--------|------|----------|------------------|---------------------|
| `LAW.statute_search` | 法规检索 | `yuandian_rh_fg_search` | 同类法规检索工具 | 法规检索类 |
| `LAW.statute_detail` | 法规详情(效力状态) | `yuandian_rh_fg_detail` | 同类法规详情工具 | 法规详情类 |
| `LAW.article_search` | 法条关键词检索 | `yuandian_rh_ft_search` | 同类法条检索工具 | 法条检索类 |
| `LAW.article_detail` | 法条详情(名+条号+日期) | `yuandian_rh_ft_detail` | 同类法条详情工具 | 法条详情/识别类 |
| `LAW.semantic_search` | 法条/法规语义检索 | `yuandian_law_vector_search` | 同类语义检索工具 | 语义检索类 |
| `LAW.case_search` | 案例检索(普通+权威) | `yuandian_rh_ptal_search`/`rh_qwal_search` | 同类案例检索工具 | 案例检索类 |
| `LAW.case_detail` | 案例详情 | `yuandian_rh_case_details` | 同类案例详情工具 | 案例详情类 |

> 运行时探测：`qw_mcp_list keyword=law/yuandian/元典/pkulaw/法宝`。具备 `LAW.citation_verify`（元典 hall_detect / 北大法宝法条识别与溯源）时，`/法律核验` 优先走该快速通道一次性核验全文。

### 文档解析后端（输入侧，多后端可选 + 灾备）

> 任何技能收到用户上传的 PDF/图片/Word/PPT 时，先解析为结构化 Markdown 再进入分析/起草。

| 项目 | 说明 |
|------|------|
| **用途** | 将上传文档解析为结构化 Markdown（含表格、公式、标题层级） |
| **使用技能** | 全套件（任何 skill 接收到上传文件时自动触发）|
| **可选后端** | 任何能提供 `PARSE.to_markdown` 能力的 MCP（MinerU / 阿里云百炼 / markdownify / 自部署 marker/docling 等） |
| **接入方式** | 用户通过 `qoder.md`「文档解析」字段配置首选后端，或首次调用时从探测到的可用后端中选择 |
| **降级链** | 首选后端 → 其他已连接后端 → 内置 pypdf（纯文字 PDF）→ fitz 视觉降级（标 `[视觉降级-待人工复核]`）→ 内置 Read。全部失败则停下报告，绝不臆造文档内容 |

### Markdown→Word 转换后端（输出侧，可选）

法律核验放行后，把 Markdown 文书一键转为 .docx Word 文件。它是一类**能力**（`DOCX.md_to_docx`），由用户在已连接的工具/CLI/MCP 中自选；首选写入 `qoder.md`「Word 转换/首选后端」字段；首选不可用时降级为仅交付 Markdown 并提示用户在本机手动转换。

| 项目 | 说明 |
|------|------|
| **用途** | 将文书 skill 输出的 Markdown 文书渲染为符合中文法院提交习惯的 .docx Word 文件 |
| **使用技能** | 所有文书 skill（主诉诉状/被诉答辩状/证据目录/质证意见/庭审提纲/代理词/程序性文书系列/新法解读/法律研究报告 等）|
| **核心价值** | md 优先 + Word 终交付；preset=legal 一键套用法院提交格式（标题加粗居中、正文宋体/仿宋、行距固定值、页边距按法院习惯）|
| **接入方式** | 用户通过 `qoder.md`「Word 转换/首选后端」字段配置，或在 `/法律核验` 放行后首次询问"是否转 Word？用什么后端转？"并写回 |
| **可选后端** | 任何能提供 `DOCX.md_to_docx` 能力的工具/CLI/MCP（含本地 python-docx 模板脚本、社区 md→docx CLI、MCP 服务等均可）|
| **降级方案** | 首选不可用 → 仅交付 Markdown 并提示用户在本机手动转换；最后兜底仅交付 Markdown |

## 连接器组合建议

### 最小配置（个人使用）

无需任何连接器。所有技能以本地模式运行，法条标注 [L4-法条待验证]，用户手动核实。

### 推荐配置（日常诉讼工作）

- **法律检索后端**（元典或北大法宝任一，法条+案例核验能力；两家都配则灾备）
- **工商信息后端**（企查查或同类，当事人核验+财产调查）
- **协作平台**（`COLLAB.*`，钉钉或飞书任一，案件管家团队协同）

三个连接器即可覆盖日常诉讼与案件管理的主要自动化需求。

### 完整配置（追求最大效率）

全部连接器接入：法律检索后端（元典 + 北大法宝，互为灾备）+ 工商信息后端（企查查/同类）+ 全网搜索（SEARCH.*）+ 文档解析（PARSE.*）+ 协作平台（COLLAB.*，钉钉全家桶或飞书全家桶）+ qmind 云端知识库。

## 降级标注规范

当连接器不可用导致能力降级时，技能输出中应统一标注：

```
[L4-{具体类别}待验证]
```

类别包括：
- `[L4-法条待验证]`：法条/法规引用未经自动核验
- `[L4-案例待验证]`：案例引用未经自动核验
- `[L4-主体信息待验证]`：当事人工商信息未经权威数据源核验
- `[L4-法院待确认]`：受理法院名称待人工确认
- `[L4-知识库不可用]`：知识库载体不可用，仅对话内回答
- `[本地模式-协作平台未连接]`：案件管家降级到本地文件模式

同时在输出文件顶部附加降级提示：
> [降级提示] {连接器名称}未连接：{具体影响说明}。如需启用完整功能，请前往 QoderWork → 设置 → 连接器 配置对应 MCP 服务。

## 连接器配置方式

本套件的 MCP 连接器通过 QoderWork 平台统一管理，无需在插件内单独配置。

**检查当前集成状态**：运行 `/冷启动访谈 --check`

**新增连接器**：在 QoderWork 设置 → 连接器 中搜索并启用对应 MCP 服务。
