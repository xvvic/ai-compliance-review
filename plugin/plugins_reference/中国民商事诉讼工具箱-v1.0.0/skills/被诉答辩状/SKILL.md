# 被诉答辩状

生成民商事被动防御方答辩文书，覆盖**诉讼程序**（一审答辩状、二审上诉答辩状、再审答辩状）与**仲裁程序**（仲裁答辩状、仲裁反请求答辩状）五种场景。

## 设计原则

> 要件攻防分析已对案件进行了系统论证，答辩状聚焦于**表达**，不重复发挥。
> 答辩状的核心价值是将防御性分析结论转化为结构严谨、层次分明的书面反驳。

## 适用边界

**本skill处理范围**：实体抗辩 + 部分非管辖类程序抗辩（诉讼时效/主体适格/重复起诉/必要共同诉讼遗漏）+ 证据层面反驳。

**不在本skill处理**：
- **管辖权异议** → 使用 `/程序性文书系列` 的管辖权异议家族（`references/02-管辖权异议.md`）。管辖异议在程序上需另文提出（答辩期内单独提交《管辖权异议申请书》），与答辩状是两份独立文书，二者不混编。
- 若被告同时主张管辖异议+实体抗辩，先以管辖异议优先处理（人民法院对管辖异议作出裁定前不展开实体审理）；管辖异议被驳回后再启动本skill撰写实体答辩状。

## 前置条件

在调用本技能前，必须确保以下材料已经准备完毕：

| 序号 | 材料 | 说明 |
|------|------|------|
| 1 | 原告起诉状 | 对方提交的起诉状/上诉状/再审申请书/仲裁申请书/仲裁反申请书原文 |
| 2 | 原告证据目录 | 对方提交的证据清单 |
| 3 | 案件事实梳理 | 经整理的时间线与关键事实（我方视角） |
| 4 | 要件攻防分析 | 包含抗辩事由、要件反驳的系统论证 |

如缺少上述材料，应提示用户先完成相应步骤，不得在缺失关键输入的情况下强行生成。

## 工作流程

### Step 1: 接收与校验材料

1. 接收用户提供的起诉状、证据目录、事实梳理、攻防分析
2. 检查四项材料的完整性：
   - 原告起诉状是否完整（含诉请和事实理由）
   - 原告证据目录是否包含证据名称与证明对象
   - 案件事实梳理是否有时间线和关键事实
   - 要件攻防分析是否包含防御策略和抗辩论证
3. 如材料不完整，列出缺失项并请用户补充

### Step 2: 识别程序类型与答辩场景

根据以下特征判断答辩场景：

**一审答辩状**（诉讼）：
- 收到一审起诉状，在答辩期内提交
- 当事人称谓：原告/被告（答辩人）
- 答辩期限：收到起诉状副本之日起15日内

**二审答辩状（上诉答辩）**（诉讼）：
- 对方提起上诉，我方作为被上诉人答辩
- 当事人称谓：上诉人/被上诉人（答辩人）
- 核心任务：维护一审判决正确性
- 答辩期限：收到上诉状副本之日起15日内

**再审答辩状**（诉讼）：
- 对方申请再审，我方作为被申请人答辩
- 当事人称谓：再审申请人/被申请人（答辩人）
- 核心任务：论证不符合再审法定事由

**仲裁答辩状**（仲裁）：
- 收到仲裁申请书，在仲裁规则规定的答辩期限内提交
- 当事人称谓：申请人/答辩人（被申请人）
- 致送：仲裁委员会（非法院）
- 开头引述案件编号："就××××仲裁委员会编号××××的案件中，对于申请人……提出的仲裁请求，现提出答辩意见如下"
- 可一并提出仲裁协议无效、仲裁庭无管辖权、仲裁员回避等程序抗辩
- 答辩书副本份数按对方当事人人数提交

**仲裁反请求答辩状**（仲裁）：
- 对方提出仲裁反请求，我方作为被反申请人答辩
- 当事人称谓：反申请人/答辩人（被反申请人）
- 开头引述："就……对于反申请人……提出的反请求事项，现提出反请求答辩意见如下"
- 致送：同一仲裁委员会

### Step 3: 构建实体防御体系（双层结构）

答辩状采用双层防御结构（管辖异议不在此处处理，见"适用边界"）：

#### 第一层：非管辖类程序性抗辩

> 注意：管辖权异议**不在**本skill处理。如有管辖异议，请使用 `/程序性文书系列`（管辖权异议家族）单独出具《管辖权异议申请书》。

仅在涉及以下非管辖程序事项时纳入答辩状：
- 诉讼时效届满（属实体性抗辩，但通常前置主张以避免程序拖累）
- 除斥期间届满
- 原告主体不适格 / 被告主体不适格
- 重复起诉（"一事不再理"）
- 遗漏必要共同诉讼参加人
- 仲裁条款排除诉讼管辖（以仲裁协议为由的不予受理抗辩，与管辖异议有交叉，但其法律效果是诉讼路径整体不当，可在答辩状中作为前置抗辩主张；同时建议另行提交管辖异议）

#### 第二层：实体性抗辩（核心）

从要件攻防分析中提取实体防御要点：
- 请求权基础不成立（法律适用错误）
- 构成要件不满足（逐要件反驳）
- 抗辩权成立（同时履行抗辩、先诉抗辩、不安抗辩等）
- 权利消灭事由（清偿、抵销、免除、混同、合同解除）
- 权利障碍事由（合同不成立、无效、可撤销）
- 免责事由（不可抗力、正当防卫、紧急避险）

#### 第三层：证据层面反驳

针对原告证据进行系统反驳：
- 证据真实性质疑
- 证据关联性否认
- 证据合法性挑战
- 证明力不足的论证
- 我方反证的提出

### Step 4: 撰写答辩状

> 📎 加载 `references/templates.md` 作为答辩状格式模板（含各审级标准结构）。

按以下结构生成答辩状正文：

#### 4.1 标题

- 一审：`民事答辩状`
- 二审：`民事答辩状`（上诉答辩）
- 再审：`再审答辩状`

#### 4.2 答辩人与对方当事人信息

**答辩人（自然人）**：
```
答辩人：[姓名]，[性别]，[民族]，[出生日期]，住所地：[地址]。
身份证号码：[号码]
联系电话：[电话]
```

**答辩人（法人/非法人组织）**：
```
答辩人：[名称]，统一社会信用代码：[代码]，住所地：[地址]。
法定代表人：[姓名]，职务：[职务]。
联系电话：[电话]
```

#### 4.3 答辩引言

```
答辩人因与[对方当事人名称][案由]一案，现针对[原告/上诉人/再审申请人]
的[起诉状/上诉状/再审申请书]，提出答辩意见如下：
```

#### 4.4 答辩意见正文

每一答辩点的撰写结构遵循"**回应 + 辩驳 + 结论**"三部分：

```
一、关于[对方主张的具体问题]

[回应]：[对方主张/观点的客观转述]
[辩驳]：[我方反驳的事实依据和法律依据]
[结论]：[该项主张不能成立/应予驳回的明确结论]
```

**排列顺序**：
1. 非管辖类程序性抗辩（如时效/主体/一事不再理，如有）排在最前
2. 核心实体抗辩居中（按对方诉请顺序逐项回应）
3. 证据层面反驳在后
4. 总结性意见在末尾

> 提醒：管辖权异议须另行提交《管辖权异议申请书》，不写入本答辩状。

#### 4.5 答辩结论

```
综上所述，[原告/上诉人/再审申请人]的[诉讼请求/上诉请求/再审申请]
缺乏事实和法律依据，请求贵院依法[驳回原告的诉讼请求/驳回上诉维持原判/
驳回再审申请]。
```

**二审特别结论**：
```
一审判决认定事实清楚、适用法律正确，请求贵院驳回上诉、维持原判。
```

**再审特别结论**：
```
再审申请人的申请不符合《民事诉讼法》第二百一十一条规定的再审事由，
请求贵院依法驳回再审申请。
```

#### 4.6 此致

```
此致
[管辖法院全称]
```

#### 4.7 落款

```
此致
[管辖法院全称]

答辩人：[签名/盖章]
[年] 年 [月] 月 [日] 日
```

> ⚠ 落款格式规则：使用纯文本，**禁止使用任何 HTML 标签**（如 `<div>`、`&nbsp;`、`<br>` 等）。Word 转换后端（DOCX.md_to_docx） 会将 HTML 标签当作文本输出导致乱码。右对齐效果由 Word 模板的段落格式控制，不在 Markdown 中实现。

### Step 5: 输出文档（md 优先）

> **⛔ 交付前必过闸门**（见 qoder.md「法律核验闸门」）：核验对象为答辩状全文。

**默认交付 Markdown**：核验放行后先产出 md 成品（写入输出目录并回链），随后询问用户「是否转 Word（Word 转换后端（DOCX.md_to_docx, preset=legal））」，确认才转 .docx。

> **输出文件命名**：`{案件简称}_答辩状_{日期}.md`（二审为 `_上诉答辩状_`，再审为 `_再审答辩状_`）。Word 版同名 `.docx`。

#### 转 Word 排版规范

> 使用 Preset A（正式文书），详见 qoder.md「文书 Word 排版规范」。

## 答辩策略选择指南

根据案件实际情况，选择合适的答辩策略组合：

| 案件态势 | 推荐策略 |
|----------|----------|
| 涉及管辖瑕疵 | **不写入答辩状**，单独走 `/程序性文书系列`（管辖权异议家族） |
| 程序有非管辖类瑕疵（时效/主体等） | 优先非管辖程序抗辩，实体答辩备选 |
| 实体权利明显不成立 | 聚焦要件不满足的论证 |
| 事实有争议 | 强化证据反驳，动摇对方举证 |
| 综合防御 | 程序+实体+证据三层全面展开 |
| 部分承认部分否认 | 明确承认范围，集中火力反驳争议部分 |

## 质量检查清单

生成答辩状后，自查以下要点：

- [ ] 诉讼阶段判断是否正确（一审/二审/再审答辩）
- [ ] 当事人称谓是否与诉讼阶段匹配
- [ ] 是否逐项回应了对方的诉讼请求/上诉请求
- [ ] 每一答辩点是否遵循"回应+辩驳+结论"结构
- [ ] **未将管辖异议混入答辩状**（如有管辖异议应另案提交）
- [ ] 双层防御体系（程序非管辖类 + 实体 + 证据反驳）是否合理运用
- [ ] 法条引用是否完整且格式正确
- [ ] 答辩结论是否明确（驳回诉请/驳回上诉/驳回再审）
- [ ] 二审答辩是否围绕维护一审判决正确性展开
- [ ] 再审答辩是否针对法定事由逐项反驳
- [ ] 排版是否符合规范要求

## 注意事项

1. **不重复分析**：答辩状应当是攻防分析结论的"表达层"，避免展开过于详细的新论证
2. **回应全面**：对方的每项诉请/上诉理由都应有对应回应，不可遗漏
3. **语言风格**：正式、精炼、逻辑清晰，反驳有力但不带情绪化表达
4. **策略性沉默**：对于无需反驳或反驳可能暴露弱点的问题，可选择不正面回应
5. **留白策略**：先尝试通过 MCP 自动填充（工商信息后端查职务、案件管家查法院/案号），仅在后端均不可用时才使用 `[待填写]` 占位并提示用户补充
6. **避免自认**：答辩中注意不要无意中承认对己不利的事实

## 下一步建议

答辩状完成后，根据案件进展可继续：
- **证据质证** → `/质证意见` 对原告证据逐项出具三性意见
- **举证准备** → `/证据目录` 编排被告方证据清单
- **庭审准备** → `/庭审提纲` 生成庭审应对预案和法庭询问清单
- **管辖异议** → 如需提管辖异议，使用 `/程序性文书系列`（答辩期内独立提交）

## Verification

生成完成后确认：
1. 文档结构完整（标题至落款无遗漏）
2. 诉讼阶段与答辩类型匹配
3. 核心论证来源于要件攻防分析（非凭空发挥）
4. 对方每项主张均有对应回应
5. Word文档成功生成且排版符合规范
6. 后端校验结果已标注（通过/待校验）
