# 程序性文书系列 · 路由中枢

> 民商事诉讼"程序性文书"统一起草入口。多数程序性文书骨架相同（标题→受文法院→当事人→请求事项→事实和理由→此致→落款→附件），差异集中在请求事项与法律依据。本技能负责**识别文书类型 → 加载对应家族子规范 → 按规范起草**，把上下文占用控制在"路由表 + 单个家族文件"。

---

## 0 | 工作原则（务必先读）

1. **不要把八个家族文件一次性全读进来。** 先用第 2 节路由表判定属于哪个家族，再 `Read` 该家族对应的 `references/0X-*.md` 单个文件，连同 `references/00-通用规范.md` 一起执行。
2. **法条必须现行有效且经核验。** 本系列内嵌的条文号以"民诉法 2023 修正（2024-01-01 施行）/ 民诉法解释 法释〔2022〕11 号"为准。写入正式文书前，按 `00-通用规范.md` 的"法条核验闸门"用法律检索后端（元典/北大法宝，运行时自动适配+灾备，见 qoder.md「外部能力后端」）的 `LAW.statute_detail`/`LAW.article_detail` 复核条文号与原文，校验不可用时标注「[L4-法条待验证]」。
3. **第八家族不是填空题。** 实现担保物权、撤销仲裁裁决、第三人撤销之诉、执行异议之诉具有较高论证深度，须充分收集材料、深度调用法律检索后端（法条+案例），实体论证要充分，参见 `references/08-其他程序类.md`。
4. **管辖权异议已完整并入本系列。** 路由到第二家族时，直接加载 `references/02-管辖权异议.md` 起草（含材料解析、工商信息核验、管辖规则分析、法条校验、撰写、Word 输出全流程），不再转交外部技能。
5. **输出为可提交法院的 Word 文书**（经 `Word 转换后端`（DOCX.md_to_docx），preset=legal 生成），保存到输出目录并回链。

> ⛔ 交付前必过闸门（见 qoder.md「法律核验闸门」）：核验对象为程序性文书全文。生成 Word 前，先调用 `/法律核验` 对全文法规/法条/案例引用逐条核验，判定全为「准确/准确(有省略)」才放行；出现「时效风险/语义不一致/未命中」须修正后重新核验。法律检索后端不可用时按降级标注「[L4-法条待验证]」并提示人工复核，不得静默跳过。

## MCP 预检

执行前调用 `qw_mcp_list` 探测以下后端是否可用：
- 法律检索：关键词匹配 `law`/`yuandian`/`pkulaw`/`法宝` → 映射 LAW.* 能力
- 工商信息：关键词匹配 `qcc`/`企查查`/`天眼查`/`biz` → 映射 BIZ.* 能力

**探测结果处理**：
1. **全部可用** → 静默进入工作流。
2. **部分不可用** → 用 `AskUserQuestion` 告知用户："检测到{不可用后端名}尚未连接。{该后端的作用}。建议前往 QoderWork → 设置 → 连接器 中配置。是否暂不配置、以降级模式继续？"
   - 用户选择配置 → 等待配置完成后重新执行
   - 用户选择继续 → 降级执行，对应位置标注 L4 标签
3. **全部不可用** → 同上询问，降级标注 `[L4-法条待验证]`/`[L4-主体信息待验证]` 等。

> 探测协议与降级规则详见 qoder.md「MCP 预检协议」。

### 2.1 易混淆消歧

- "解除保全" → ③；"撤回保全申请" 属于撤回行为，亦在 ③ 处理。
- **仲裁保全**：仲裁财产保全申请书致送仲裁委员会（由仲裁委转交法院执行），归 ③。"诉前/仲裁前保全"致送有管辖权的法院，同归 ③。
- "证人出庭" → ⑤（非 ⑥）；"专家辅助人/有专门知识的人出庭" → ⑥（与鉴定同组，因其针对鉴定意见与专业问题）。
- "委托书/身份证明" → ①；本家族提供**通用空白范本与填写规则**。
- "执行异议"（对执行行为/执行标的，向执行法院提，裁定式）→ ⑦；"执行异议之诉"（异议被裁定驳回后另行起诉，判决式）→ ⑧。
- "撤销仲裁裁决" vs "不予执行仲裁裁决"：前者向仲裁委所在地中院申请（⑧）；后者在执行阶段提出（执行抗辩，归 ⑦，并在 ⑧ 交叉指引）。
- **仲裁管辖权异议**：由被申请人向仲裁委员会提出（请求驳回申请人仲裁申请），归 ②；与法院管辖权异议共用家族，但致送对象和法律依据不同（仲裁法第20条）。
- **增加/变更/放弃仲裁请求**：归 ④，结构同诉讼版，致送改为仲裁委员会。

## 3 | 标准执行流程

```
用户请求
  → 第2节路由判定家族（必要时1问澄清）
  → Read references/00-通用规范.md（共用：信息清单/称谓/法条核验闸门/输出规范）
  → Read references/0X-该家族.md（该家族专属：子文书清单/请求事项写法/专属要件/法律依据/附件）
  → 收集案件信息（缺则按清单提问；可读取已上传材料/案件工作空间）
  → 法条核验闸门：法律检索后端复核条文号与原文（不可用标[L4-法条待验证]；第七家族另需案例检索）
  → 起草（套用骨架 + 家族专属请求事项与事实理由）
  → ⛔ 输出前必过闸门：调用 /法律核验 逐条核验全文法规/法条/案例引用，判定全为「准确」才放行（不可用按降级标[L4-法条待验证]并提示人工复核）
  → Word 转换后端（DOCX.md_to_docx, preset=legal） 生成 → 保存输出目录 → 回链 + 提交提示（份数、签章、附件、期限）
```

## 5 | 执行规则

1. **路由克制**：1–2 轮内完成家族判定，不过度追问。
2. **按需加载**：只读取命中的家族文件 + 通用规范，严禁全量读取。
3. **核验前置**：任何写入文书的法条，先过核验闸门；校验不可用标「[L4-法条待验证]」。
4. **不杜撰**：缺失的案件信息向用户索取，不臆造当事人、案号、金额、财产信息。
5. **文件安全**：改写用户已有文书前先备份；最终成果输出到输出目录并回链。
