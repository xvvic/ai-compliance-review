# 00 · 通用规范（所有家族共用）

> 起草任何程序性文书前，连同命中的家族文件一起执行本规范。本文件定义：信息清单、当事人称谓规则、文书通用骨架、法条核验闸门、Word 输出与提交清单。

---

## A | 通用信息清单（缺则向用户索取，勿臆造）

1. **受理/管辖法院全称**（保全/执行可能为财产所在地或被申请人住所地法院）。
2. **案号**（已立案的诉讼内文书必填："（××××）×××民初/民终/执×号"）。
3. **申请人/异议人/委托人**信息：
   - 自然人：姓名、性别、出生日期、民族、工作单位与职务或职业、住所、联系方式、身份证号（部分文书需）。
   - 法人/其他组织：名称、住所、统一社会信用代码；法定代表人/主要负责人姓名与职务、联系方式。
4. **被申请人/相对方**同上口径的基本信息。
5. **文书专属要素**（见各家族文件，如保全标的、举证事项、被执行人财产线索、鉴定事项等）。
6. **委托诉讼代理人**（如有）：姓名、所在律所/单位、是否特别授权。

> 当事人企业信息（名称、统一社会信用代码、法定代表人姓名与**职务**、住所）可用**工商信息后端（企查查/同类，见 qoder.md「外部能力后端」）** 核验：先用 `BIZ.company_registration_info` 获取基础工商信息，再用 `BIZ.key_personnel` 查询主要人员以自动填充法定代表人职务，失败标注「[L4-主体信息待验证]」。**受理法院名称与案号**优先从案件管家（`COLLAB.sheet_query`，见 qoder.md「案件管理」配置）以当事人名称为 keyword 查询自动填入，台账不可用时从起诉状/来文中提取，仍无法获取则标注「[L4-法院待确认]」。财产线索可调用 `/财产线索调查` 技能。用户上传的起诉状/合同/裁定书等文件，可用文档解析连接器（如已配置）或 QoderWork 内置 Read 直接读取，避免手动转录。

## B | 当事人称谓规则（按文书所处程序确定）

| 程序/文书 | 我方称谓 | 对方称谓 |
|-----------|---------|---------|
| 诉前保全 | 申请人（利害关系人） | 被申请人 |
| 诉讼中各类申请 | 申请人 / 当事人（一审：原告/被告；二审：上诉人/被上诉人） | 被申请人 / 对方当事人 |
| 解除保全 | 申请人（被保全人） | 被申请人（申请保全人） |
| 执行程序 | 申请执行人 | 被执行人 |
| 执行异议 | 异议人（当事人/利害关系人/案外人） | （列明被执行人、申请执行人） |
| 第三人撤销之诉 | 原告（第三人） | 被告（原生效裁判的原被告） |
| 执行异议之诉 | 原告（案外人或申请执行人） | 被告（申请执行人/被执行人） |
| 实现担保物权 | 申请人（担保物权人/所有权人等） | 被申请人（担保人/债务人） |
| 撤销仲裁裁决 | 申请人 | 被申请人 |

落款：自然人签名（捺印）；法人/其他组织盖公章 + 法定代表人/负责人签字。日期用"××××年××月××日"。

## D | 法条核验闸门（强制，写入正式文书前执行）

各家族文件中内嵌的条文号已按"民诉法 2023 修正（2024-01-01 施行）/ 民诉法解释 法释〔2022〕11 号"标注，但**仍须在生成前用法律检索后端（元典/北大法宝，运行时自动适配+灾备，见 qoder.md「外部能力后端」）复核**，步骤：

1. **现行条文核验**：调用法律检索后端取现行原文与条文号：
   - `LAW.statute_detail`（fgmc="中华人民共和国民事诉讼法"，refer_date 取当日）或
   - `LAW.statute_search`（fgmc + keyword + sxx="现行有效"）。
   司法解释同法用 fgmc="最高人民法院关于适用《中华人民共和国民事诉讼法》的解释"。
2. **一致性判定**：条文号/原文不一致时，**以法律检索后端现行版为准**，修正家族文件中引用并据实落笔。
3. **MCP 不可用降级**：法律检索后端不可用时，对应法条标注「[L4-法条待验证]」（或 [L4-法条待验证]），并附待校验清单，继续完成文书主体，提示用户人工核实。
4. **⛔ 输出前必过 `/法律核验` 闸门**：文书定稿、调用 `Word 转换后端`（DOCX.md_to_docx） 之前，统一调用 `/法律核验` 对全文法规/法条/案例引用做最终逐条核验。判定全为「准确/准确(有省略)」才放行进入 E 节输出；出现「时效风险/语义不一致/未命中」须按报告修正后重新核验。此闸门不可绕过、不可静默跳过。

> 提示：第七家族（其他程序类）还须用法律检索后端的案例检索（`LAW.case_search` 权威案例 / `LAW.case_semantic` 语义检索）检索支撑类案。

## E | Word 输出与提交清单

1. 调用 **`Word 转换后端`（DOCX.md_to_docx）（preset=legal）** 生成正式 Word；正文宋体/仿宋、标题加粗居中、行距固定值，符合法院提交习惯。
2. 保存到输出目录（无选定文件夹时存 `outputs/`），文件名形如 `〔案号或当事人〕+文书名称.docx`，完成后用 `file://` 回链。
3. 文末向用户给出**提交提示**：
   - 需提交份数（通常正本 + 按对方当事人数的副本）；
   - 签章要求（自然人签名捺印 / 单位盖章 + 法代签字）；
   - 必备附件是否齐备（裁定书、证据、清单、担保材料、身份/授权材料）；
   - 关联期限（如诉前保全 30 日内起诉、复议 / 上诉期、举证期限），请自行核算并提醒用户登记核对（本套件无独立期限技能）。
4. 不杜撰署名与日期，留占位由用户确认后填写。
