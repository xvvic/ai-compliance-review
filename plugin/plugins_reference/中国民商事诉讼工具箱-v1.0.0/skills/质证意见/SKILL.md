# 质证意见

## 概述

本技能用于生成民事诉讼中的质证意见书。根据对方提交的证据材料，结合案件事实和要件攻防分析，逐项对证据的**"三性"（真实性、合法性、关联性）**进行审查。**证明力（能否达到证明目的、孤证、推论跳跃等）并入关联性分析，不单独作为一个维度**——理由是：证明力本质上是证据与待证事实关联强度的延伸判断，归入关联性更符合"证明力＝关联到何种程度"的逻辑，也避免与关联性重复评价。

## 前置条件

执行本技能前，必须确认以下材料已就绪：

| 序号 | 材料 | 必要性 | 说明 |
|------|------|--------|------|
| 1 | 对方证据目录 | 必需 | 含证据编号、名称、证明目的 |
| 2 | 对方证据册 | 必需 | 证据原文或扫描件 |
| 3 | 案件事实梳理 | 必需 | 本案时间线及关键事实 |
| 4 | 要件攻防分析 | 必需 | 各请求权基础的构成要件及攻防态势 |

如任一材料缺失，提示用户补充后再行质证。

## MCP 预检

> 预检协议详见 qoder.md「MCP 预检协议」。本技能使用后端：LAW.*（质证中引用法条的自动校验）。
> 不可用时降级标注：`[L4-法条待验证]`。

## 工作流

### Step 1: 材料解析与格式预检

1. 解析对方证据目录，提取证据清单（编号、名称、证明目的）
2. 逐项比对证据册，确认各项证据的实际内容
3. 回顾案件事实梳理与要件攻防分析，标记各证据关联的争议焦点

**格式预检**：先核对证据清单字段是否齐备，缺失字段会影响实质质证，应先提示补全或在质证意见中标注。

| 字段 | 有无？ | 缺失影响 |
|------|--------|----------|
| 编号 | | 无法逐项对应 |
| 证据名称 | | 指代不清 |
| 证据来源 | | 影响合法性判断 |
| 证明目的（证明内容） | | 无法判断关联性 |
| 证据形式（书证/物证/电子数据/证人证言等） | | 影响审查要点选择 |
| 是否为原件/原物 | | 影响真实性判断 |

### Step 2: 逐项质证分析（内部思考顺序）

> 📎 加载 `references/evidence-review-guide.md` 作为三性审查框架与写作顺序参考。

对每份证据，按以下**分析顺序**进行内部思考（此顺序用于分析，非写作顺序）：

#### 2.1 关联性分析（含证明力，优先判断）

关联性分析吸收原"证明力"维度，分两个层次：

**第一层·有无关联（关联性本身）**
- 该证据与本案争议焦点是否相关？
- 与对方主张的待证事实之间有无逻辑联系？
- 是否属于与本案无关的"干扰证据"？

**第二层·关联强弱（证明力，并入关联性）**
- 即使与待证事实相关，该证据能否达到对方的证明目的？
- 是否存在证明力不足的情形（孤证、间接证据、推论跳跃、只能证明部分事实等）？
- 综合全案证据体系评估其证明效果（是否需其他证据佐证、是否形成完整证据链）

> 写作时这两层统一写在"关联性"项下，先答有无关联，再答能否达到证明目的，不另起"证明力"段。

#### 2.2 合法性分析

- 证据形式是否合法（来源、取得方式）？
- 是否存在以下违法取证情形：
  - 严重侵害他人合法权益（偷录私密空间等）
  - 违反法律禁止性规定
  - 严重违背公序良俗
- 举证形式是否符合要求（原件规则等）

#### 2.3 真实性分析

- 证据形式真实性（是否伪造、变造、篡改）
- 证据内容真实性（内容是否如实反映事实）
- 关键假定：**除非用户明确确认对方持有原件，否则一律假定对方无法提供原件**
- 电子数据的专项审查（存储介质、完整性、哈希值等）

#### 2.4 三态判断与边界标记

每项三性结论落入三态之一，**绝不在不确定时沉默地下定论**：

- **不予认可（明确异议）**——有明确依据可质疑，直接写明异议及理由。
- **无法确认（⚠️ 待定）**——边界模糊（如复印件未见原件、录音取得方式临界、关联性边界不清）：写"无法确认"并**同时记录支持与反对两个方向的理由及待查问题**，便于代理律师在庭前定夺，而非替其拍板。
- **认可（无异议）**——无可争辩或属对方自认事实（见 Step 3），明确认可以维护质证诚信。

> **倾向可恢复的错误**：对我方而言，盲目"全面否认"可能损害诚信、引发法官反感（不可逆）；保守地认可客观事实、把火力集中在关联性/证明力上，是可回旋的策略。存疑时倾向后者，并交律师定夺。

### Step 3: 自认事实识别

结合要件攻防分析，判断对方某份证据是否属于"对方自认的对其不利事实"：

- 如果是，主动提示用户：
  > 💡 **提示**：证据[编号]中[具体内容]属于对方自认的对其不利事实，建议考虑认可其真实性和合法性，仅从关联性（证明力）角度发表意见或予以援用。

### Step 4: 撰写质证意见

**写作顺序**（实务惯例）：真实性 → 合法性 → 关联性（含证明力）

对每份证据输出：

```
证据编号：[X-X]
证据名称：[名称]
真实性意见：[认可/不认可/无法确认] + 理由
合法性意见：[认可/不认可/无法确认] + 理由
关联性意见：[认可/不认可/部分认可] + 理由（先答有无关联，再答能否达到证明目的）
综合意见：[简要总结立场]
```

### Step 5: 跨清单模式标记

逐项质证后，对整组证据做一次横向观察，在意见末尾归纳：

- 同一瑕疵是否在多份证据中重复出现（如均为复印件、均无原始载体）？
- 是否存在过度举证/重复举证（多份证据指向同一事实却均无法独立证明）？
- 对方证明目的描述是否笼统模糊，导致证据与待证事实脱节？
- 证据之间是否存在相互矛盾，可供我方利用？

模式标记有助于在综合意见中主张"对方证据体系无法形成完整证据链"。

### Step 6: 法条引用校验

> 法律检索后端探测与降级详见 qoder.md「MCP 预检协议」。

```
能力调用：LAW.article_detail / LAW.statute_detail（法条/法规校验）
检索范围：证据规则相关法条
重点法条：
- 《民事诉讼法》第66-71条（证据种类与质证）
- 《最高人民法院关于民事诉讼证据的若干规定》
- 《最高人民法院关于适用〈民事诉讼法〉的解释》第103-116条
```

> ⚠️ 如法律检索后端均不可用，法条引用标注为"[L4-法条待验证]"，提示用户自行核实条文编号及内容。

### Step 7: 输出文档（md 优先）

> **⛔ 交付前必过闸门**（见 qoder.md「法律核验闸门」）：核验对象为质证意见全文。

**默认交付 Markdown**：核验放行后先产出 md 成品（写入输出目录并回链），随后询问用户「是否转 Word（Word 转换后端（DOCX.md_to_docx, preset=legal））」，确认才转 .docx。

> **输出文件命名**：`{案件简称}_质证意见_{日期}.md`。Word 版同名 `.docx`。

## 各类证据质证要点体系

> 下表按三性组织；原"证明力"要点已并入"关联性（含证明力）"行，写作时统一在关联性项下表述。

### 一、书证

| 审查维度 | 要点 |
|----------|------|
| 真实性 | 是否原件；印章真伪；签名真伪；是否存在涂改、添加；形成时间 |
| 合法性 | 取得途径是否合法；是否属于应当由特定主体出具的文书 |
| 关联性（含证明力） | 内容是否指向本案事实；时间节点是否匹配；公文书证vs私文书证的证明力差异；单位证明是否符合法定形式而影响证明效果 |

### 二、物证

| 审查维度 | 要点 |
|----------|------|
| 真实性 | 保管链条是否完整；是否存在污染、替换可能 |
| 合法性 | 取得方式；保全措施是否合法 |
| 关联性（含证明力） | 物证特征与待证事实的逻辑联系；鉴定结论支持程度；物证的唯一性/排他性 |

### 三、视听资料

| 审查维度 | 要点 |
|----------|------|
| 真实性 | 是否原始载体；有无剪辑、拼接痕迹；录制环境辨识 |
| 合法性 | 录制是否经对方同意（需区分场景）；是否侵害隐私权 |
| 关联性（含证明力） | 内容与争议焦点的关联度；对话主体身份确认；完整性（是否截取片段）；语境还原是否影响证明效果 |

### 四、电子数据

| 审查维度 | 要点 |
|----------|------|
| 真实性 | 生成/存储/传输环境可靠性；哈希值校验；时间戳 |
| 合法性 | 取证程序（是否公证保全）；区块链存证合规性 |
| 关联性（含证明力） | 电子数据生成主体与当事人的关联；是否经公证；是否有关联证据佐证；平台数据的证明标准 |

### 五、证人证言

| 审查维度 | 要点 |
|----------|------|
| 真实性 | 证人感知能力；记忆可靠性；陈述一致性 |
| 合法性 | 证人资格（利害关系）；作证程序（是否出庭） |
| 关联性（含证明力） | 证言内容与待证事实的直接/间接关系；亲历vs传闻；单一证人vs多证人印证；品格证据 |

### 六、鉴定意见

| 审查维度 | 要点 |
|----------|------|
| 真实性 | 检材来源及保管；鉴定方法科学性 |
| 合法性 | 鉴定机构/人员资质；鉴定程序合规性；委托程序 |
| 关联性（含证明力） | 鉴定事项与争议焦点的关联；鉴定意见的确定性程度；是否存在多份鉴定冲突 |

### 七、当事人陈述

| 审查维度 | 要点 |
|----------|------|
| 真实性 | 前后陈述一致性；与其他证据的印证/矛盾 |
| 合法性 | 陈述环境（是否受胁迫等） |
| 关联性（含证明力） | 陈述内容与争议焦点的对应关系；自认规则适用；不利陈述的证明效果 |

## 质证策略提示

### 全面否认策略

适用场景：对方证据体系整体薄弱
- 逐项否认三性
- 强调证据体系无法形成完整证据链

### 选择性认可策略

适用场景：部分证据确实客观存在，但证明目的不成立
- 认可真实性、合法性，从关联性（含证明力）角度否认其证明效果
- 避免"全面否认"导致法官质疑诚信

### 证据链断裂策略

适用场景：对方依赖间接证据推理
- 认可单项证据真实性，在关联性项下否认推理逻辑
- 指出证据链条中的断裂环节

## 注意事项

1. **原件规则**：未提供原件的书证，真实性意见中必须提及
2. **禁止自认**：质证中不得对我方不利事实做出自认性表述
3. **证明力归入关联性**：不单列证明力意见；即使认可真实性、合法性，仍可在关联性项下否认其证明力（先答有无关联，再答能否达到证明目的）
4. **预留空间**：对尚未质证的证据（如后续补充证据），保留质证权利声明
5. **兼顾攻防**：质证不仅是防守，也要通过质证暴露对方举证不足
6. **存疑交律师**：边界模糊处标"无法确认"并记录两面理由，不替代理律师拍板（按证据三性的三态规则）

## 下一步建议

质证意见完成后，根据案件进展可继续：
- **庭审准备** → `/庭审提纲` 将质证要点整合到庭审应对方案中
- **代理意见** → `/代理词` 基于质证结论起草正式代理词
- **补充举证** → `/证据目录` 如需补充己方反驳证据

## 常见质证口径模板

### 真实性异议

- "对该证据的真实性不予认可。该证据系复印件/打印件，对方未提供原件供核对，无法确认其真实性。"
- "对该证据的真实性不予认可。该证据存在明显的[涂改/拼接/时间矛盾]痕迹。"
- "对该证据的形式真实性不持异议，但对其内容真实性不予认可。[具体理由]"

### 合法性异议

- "对该证据的合法性不予认可。该证据系通过[具体违法方式]取得，违反了[具体法律规定]。"
- "对该证据的合法性不予认可。该单位证明未附出具人身份信息及联系方式，不符合《民事诉讼证据规定》第92条的形式要求。"

### 关联性异议（含证明力）

- "对该证据的关联性不予认可。该证据所涉事项为[内容]，与本案争议焦点[焦点]无关。"
- "对该证据的关联性不予认可。该证据形成时间为[时间]，与本案所涉[时间段]的事实不具有时间上的关联。"
- "对该证据的真实性、合法性不持异议，但其与对方证明目的不具有关联性。对方主张[证明目的]，但该证据仅能证明[实际证明事项]，不能推导出[对方结论]。"
- "该证据属于孤证，在无其他证据佐证的情况下，不足以证明对方主张的[待证事实]。"

## 本技能不做什么

- **不替代理律师做临界判断**：边界模糊处标"无法确认"并列两面理由，交律师定夺，不擅自下结论。
- **不单列证明力维度**：证明力一律并入关联性表述，避免重复评价。
- **不对我方不利事实作自认**：质证口径严守防御立场。
- **不保证认可项的绝对正确**：质证意见是代理工作底稿，最终由律师对口径负责。

## 错误处理

- 证据目录与证据册不一致 → 提示用户确认
- 证据编号缺失或重复 → 提示用户核对
- 后端服务不可用 → 按 qoder.md 降级规则标注，继续完成质证意见主体。
- 证据类型无法识别 → 询问用户确认证据种类
- 证据数量过多（超过30份） → 建议分组质证，按争议焦点归类
