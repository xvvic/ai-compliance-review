# 案件管家 · 案件全生命周期管理

> 一站式管理案件从**收案 → 在办 → 结案 → 归档**的全生命周期。集建档登记、日常跟进更新、期限计算与提醒、组合全景、结案归档于一体。协作平台结构化表格（`COLLAB.sheet_*`）可作为团队协同载体（首选），无连接器时零依赖本地运行 `[本地模式-协作平台未连接]`。
>
> **设计哲学**：最小干预、自动沉淀——在日常办案交互中，台账随工作推进而自动更新，用户无需刻意"维护台账"。
>
> **律师/法务双模式**：读取 `qoder.md`「角色」字段自动切换。

   - **首选**：协作平台结构化表格（`COLLAB.sheet_*`，运行时探测钉钉/飞书/同类）。
   - **降级** `[本地模式-协作平台未连接]`：本地 `litigation-portfolio.json`（组合账本）+ 每案 `matters/<slug>/matter.md` + `history.md`。
3. 子模式分发：`建档 | 跟进 | 期限 | 全景 | 结案 | 归档`。无参数 → 询问意图。

## 1 | 案件跟踪记录表（最小必要字段）

> 以下字段设计适配所有法务和律师场景，不预设特定公司或业务模式。

| 分组 | 字段 | 类型/说明 |
|------|------|-----------|
| 基础信息 | 案件编号 caseId、案件名称 caseName、案由 cause、我方当事人 ourParty、我方地位 ourRole、对方当事人 counterparty、标的额 amount、审理机关 court、案号 docketNo | 文本/数值/单选 |
| 归属 | 主办人 handler、协办 coHandler、所属业务线 bizLine`[法务]`、委托人 client`[律师]` | 文本 |
| 生命周期 | 当前阶段 stage（立案/一审/二审/再审/执行/仲裁/已结案）、风险等级 risk（重大/高/中/低）、案件来源 source | 单选/文本 |
| 关键期限 | 下一期限 nextDeadline、期限描述 deadlineDesc、举证届满 evidenceDue、开庭日 hearingDate、上诉期届满 appealDue | 日期/文本 |
| 进度 | 当前进展 progress（自由文本，每次更新追加）、下一步动作 nextAction | 文本 |
| 结案 | 结案日 closedDate、审理结果 result、outcome（胜诉/部分胜诉/败诉/调解/撤诉/和解/其他）| 日期/单选 |
| 元数据 | 最后更新 lastUpdated、备注 notes | 日期/文本 |

> 用户可在冷启动访谈中自定义扩展字段（如外部律师信息、预算、利益冲突等），以上为最小通用集合。

## 2 | 办案流程（通用默认 + 用户自定义）

> 读取 `qoder.md`「办案流程」。用户未自定义时，以下通用流程生效：

**通用办案流程**（基于本套件功能生态）：

- **立案/收案**：建档登记 → 初步法律评估 → 确定策略
- **准备阶段**：案件事实梳理 → 法律关系图与时间轴 → 检索研究 → 要件攻防分析
- **文书阶段**：起诉/答辩 → 证据目录与举证 → 庭前准备（庭审提纲）
- **庭审阶段**：开庭 → 庭后代理词/补充意见
- **收尾阶段**：等判决 → 领取裁判文书 → 是否上诉/执行决策
- **结案阶段**：结案复盘 → 知识库沉淀 → 归档

> 用户提供了自定义流程时，以用户版本为准。跟进提醒默认通过 IM 推送或对话展示（最轻量）；如用户在冷启动中配置了日程提醒（`COLLAB.calendar_create`），则关键期限同步写入日历。

## 3 | 子模式

### 3.1 建档（intake）

1. 采集基础信息（案件名称、案由、当事人、标的额、法院、我方地位、案件来源、案号）。缺项标 `[待验证:事实]`。
2. **⛔ 利益冲突门禁 `[仅律师]`**：建档前须确认冲突检索已完成。法务模式跳过。
3. 写入数据源（`COLLAB.sheet_create` 或本地 JSON + `matters/<slug>/`）。
4. 生成 `matters/<slug>/matter.md`（从模板）。
5. 初始化 `matters/<slug>/history.md`（首条建档事件）。
6. **衔接**：推荐 `/案件事实梳理`、计算首批期限（如已有法院文书）。

### 3.2 跟进（update）

1. 定位案件（caseId/案件名称/案号）。
2. 审计追加：向 `history.md` 追加事件（日期+事项+来源技能），不覆盖。
3. 刷新字段：阶段、风险、下一步动作、进展摘要、`lastUpdated`。
4. 同步 Matter Profile 对应字段。
5. **重要变化触发器**：和解达成、改判、追加当事人、保全临期、风险升档——提示是否联动相关技能。

**跨技能自动同步**（供其他技能调用）：
- 任何套件技能完成案件相关工作后自动调用本子模式
- 写入格式：`{日期} {技能名称}完成：{简要成果描述}`
- 不阻塞原则：同步失败时静默降级到 history.md 追加，不中断主流程

### 3.3 期限（deadline）

> 诉讼期限计算与待办管理。解析法院文书/送达信息 → 按民诉法计算各类期间 → 生成待办。

**期限类型**（计算依据须经 `/法律核验` 复核现行条文）：

| 类型 | 期间 | 起算 |
|------|------|------|
| 举证期限 | 法院指定（一审普通≥15日） | 受理/答辩期满后 |
| 答辩期 | 15日（涉外30日） | 收到起诉状副本次日 |
| 上诉期 | 判决15日 / 裁定10日 | 送达次日 |
| 申请再审 | 6个月 | 裁判生效次日 |
| 申请执行 | 2年 | 履行期最后一日次日 |
| 管辖异议 | 答辩期内 | 收到起诉状副本起 |
| 保全冻结 | 存款1年/动产2年/不动产3年 | 裁定执行日 |

**计算流程**：
1. 采集来源（用户上传文书/粘贴/电子送达），提取送达日、文书类型、期间等。
2. 按民诉法规则计算（起算日不计入、末日节假日顺延）。
3. 经 `/法律核验` 复核条文现行有效性。
4. 回写案件对应期限字段。
5. 生成待办条目，按四档分类：🔴过期 / 🟠今日 / 🟡≤7天 / 🟢≤30天。

**查看待办**：跨案件聚合，按紧迫度展示。

**定时提醒**（可选，依赖 qoder_cron）：
- 用户在冷启动访谈中配置提醒渠道后，注册每日巡检任务
- 经 IM 推送临期/过期待办
- 如用户启用了日历集成（`COLLAB.calendar_create`），关键期限可同步写入日程

**失权警示**：所有期限输出附"本计算供参考，最终以法院文书载明与承办律师核实为准"。

### 3.4 全景（briefing）

- **单案简报**：当前阶段/风险/下一期限/近期变化/未完成事项/建议下一步。
- **组合仪表盘**：风险分布、阶段分布、临期案件、滞后案件（长期未更新）、本月结案数。
- **异常巡检**：① 高风险且临期；② 保全将到期未续封；③ 上诉期/执行期临近未决；④ `[律师]` 利益冲突待检索；⑤ `[法务]` 外部律师状态待定。
- 不做决定，只呈现事实与异常。

### 3.5 结案（close）

1. 捕获结案信息：结案日、结果、outcome、最终标的、是否上诉/执行。
2. 敞口校准：结果 vs 建档时初始预判对比。
3. 标记 stage=已结案，保留在账本（不删除）。
4. **强衔接 `/案件复盘`**：建议进入结案复盘。

### 3.6 归档（archive）

1. 整理案卷（按时间顺序与重要性分类）。
2. 生成案卷目录清单（标准装订顺序）。
3. 完整性核对，标注缺失项。
4. 移交/归档确认：
   - 律师模式：移交确认书（注意工作底稿抽出）
   - 法务模式：公司档案归档清单

## 5 | 执行规则

1. **不越权**：只管理案件记录，不起草文书、不做法律分析、不替律师/法务决策。
2. **审计可追溯**：所有更新仅追加 history，不覆盖。
3. **保密**：默认 CONFIDENTIAL，不跨案件泄露。
4. **降级透明**：连接器缺位时明确标注 `[本地模式-协作平台未连接]`。
5. **提醒克制**：默认仅通过 IM 推送或对话展示提醒，不主动创建日程或操作群组——除非用户在冷启动中明确启用日历集成。
6. **最小干预**：台账字段精简通用，不预设复杂流程，随用户交互自然沉淀。
