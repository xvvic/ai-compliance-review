# 财产线索调查

## 适用范围

本skill适用于民商事诉讼中，原告方（或有保全需求的当事人）对被告/被申请人开展财产线索调查的场景：

1. **诉前财产摸底**：评估被告偿付能力，决定是否起诉
2. **保全前置尽调**：申请诉前/诉中财产保全前，确定保全标的
3. **执行阶段线索**：判决生效后，辅助执行阶段发现被执行人财产

## 设计哲学

本报告不是传统的数据目录（按数据类型平铺 9 类信息），而是以行动为导向的保全行动指南：

- **先给结论**：报告第一页即为"保全价值速览"，律师 30 秒内可判断"有没有东西可以保"
- **按操作分类**：全部发现按保全方式归入四条通道（直接锁定 / 调查追踪 / 债权拦截 / 关联穿透），而非按数据类型归入九个章节
- **统一结构**：每个资产项统一采用"发现 → 评估 → 操作"三项结构，可直接转化为保全申请书
- **独立分析**：诉讼竞合态势作为独立章节，评估前序债权人的保全竞合风险
- **行动收尾**：以"保全操作路线图"收尾，给出即刻/诉中/执行三阶段的具体行动建议

- 全网搜索：关键词匹配 `search`/`搜索`/`web`/`AiSearch` → 映射 SEARCH.* 能力
- 法律检索：关键词匹配 `law`/`yuandian`/`pkulaw`/`法宝` → 映射 LAW.* 能力

**探测结果处理**：
1. **全部可用** → 静默进入工作流。
2. **工商信息后端不可用** → 用 `AskUserQuestion` 告知用户："工商信息 MCP 未连接，财产线索调查将降级为全网搜索公开信息，数据结构化程度较低。建议前往 QoderWork → 设置 → 连接器 中配置企查查或同类。是否暂不配置、以降级模式继续？"

## 工作流程

### 第一步：确认目标与范围

向用户确认：
1. 目标主体名称（企业名称/自然人姓名）
2. 调查目的（诉前摸底/保全准备/执行线索）
3. 已知信息（注册号/统一社会信用代码/已知财产等）
4. 保全金额参考（诉讼标的额或预估金额）
5. 穿透深度（仅目标公司 / +子公司 / +实控人控制企业 / +历史关联）

### 第二步：工商基本信息查询

调用工商信息后端获取：
- 企业注册信息（注册资本/实缴资本/经营状态）
- 法定代表人信息
- 股东信息及持股比例
- 对外投资情况
- 分支机构

**能力调用**：
- `BIZ.company_detail`（企业注册信息）
- `BIZ.shareholder`（股东信息）
- `BIZ.investment`（对外投资/实控人/受益所有人）

### 第三步：四通保全数据采集

> 📎 加载 `references/data-source-matrix.md` 作为数据源与能力映射矩阵（按保全通道聚合）。

按 `references/report-skeleton.md` 的通道顺序，依次采集四条通道的数据。每个资产项统一产出 3 块内容：
- **发现**（来自工具返回，逐条标注来源）
- **评估**（分析保全可行性 + 引用法律依据）
- **操作**（给出具体保全动作建议）

#### 通道一 · 直接锁定
已有公开数据支撑、可直接向法院申请保全措施的资产：
- 3.1 银行账户与金融线索
- 3.2 对外投资股权
- 3.3 知识产权出质质
- 3.4 动产与不动产抵押/查封

#### 通道二 · 调查追踪
存在线索但需进一步调查确认后方可实施保全的资产：
- 3.5 银行账户深挖线索
- 3.6 地址与动产线索
- 3.7 土地与在建工程
- 3.8 高价值知识产权
- 3.9 业务资质与经营现金流

#### 通道三 · 债权拦截
目标公司的到期债权或预期收益：
- 3.10 招投标项目应收账款
- 3.11 诉讼胜诉债权
- 3.12 经营合同预期收益

#### 通道四 · 关联穿透
通过追究股东责任、执行分支机构财产或揭开关联公司面纱扩大保全范围：
- 3.13 股东出资责任追究
- 3.14 分支机构财产执行
- 3.15 实控人及关联企业
- 3.16 历史关联方追踪

### 第四步：诉讼与执行竞合态势分析

独立分析目标公司在诉讼和执行领域的整体态势：
- 作为被告/被执行人：前序债权人竞合风险
- 作为原告：胜诉债权与回款追踪
- 信用恶化信号汇总（失信/限消/终本/股权冻结）
- 保全顺位与时序风险评估

### 第五步：保全操作路线图

> 📎 加载 `references/preservation-checklist.md` 作为财产保全法律依据必检清单。

将前述分析凝练为可执行的行动方案：
- 保全申请要点
- 担保策略（依据《财产保全规定》第 5 条）
- 时机与节奏建议（即刻/诉中/执行三阶段）
- 轮候冻结预案
- 善意文明执行注意事项

### 第六步：输出报告

> **⛔ 交付前必过闸门**（见 qoder.md「法律核验闸门」）：核验对象为报告引用的法规/法条。

**质量门检查（所有项必须通过才可交付）**：
- [ ] 四条通道下每个资产项都有"发现 + 评估 + 操作"三段
- [ ] "保全价值速览"的四通道汇总表已填写
- [ ] "保全优先级矩阵"四象限已填写
- [ ] "诉讼与执行竞合态势"章节完整
- [ ] "保全操作路线图"三阶段行动建议已填写
- [ ] 法条引用经校验或标注待校验
- [ ] 数据来源逐条标注
- [ ] 无数据的资产项明示"暂无公开记录"，不得编造
- [ ] 报告客观中立，未做超出证据的判断
