# 案件复盘 · 研析裁判文书与结案复盘

> 裁判文书或庭审结束后，站在**特定用户视角**输出可对外汇报的分析与复盘。三级递进、两种深度。复盘只做客观分析与经验提炼，**不预测后续案件结果、不替代律师决策**；引用法条/案例**必过 `/法律核验` 闸门**后才输出。

---

## 0 | 前置：判定层级与深度

1. 读 `qoder.md`：角色（律师/法务）、立场、风险校准、内部汇报风格、行业视角。
2. **判定复盘层级**（按用户意图/材料）：
   - **庭审复盘**：刚开完庭、有庭审记录/笔录 → 第 2 节
   - **裁判文书研析**：收到判决/裁定/调解书 → 第 3 节
   - **结案复盘报告**：案件终局、需结案总结/对外汇报 → 第 4 节（通常含第 3 节研析）
3. **判定深度模式**（关键）：
   - **自办案件（深度）**：本案在 `/案件管家` 有记录 → 自动调取台账、`history.md`、`/案件事实梳理`产物、核心过程文书（起诉/答辩/证据目录/质证/代理词/庭审提纲），站在承办视角做深度复盘。
   - **非自办 / 受限信息（降级）**：本案非自己办理或缺过程记录 → **仅基于用户提供的裁判文书与有限信息**分析，全程标注「基于受限信息」，不臆造办案过程；适合为客户/法务领导/业务部门做第三方解读。
4. 复核材料是否齐全；缺关键件标 `[证据缺口]`/`[待验证:事实]`，不臆造。

## MCP 预检

执行前调用 `qw_mcp_list` 探测以下后端是否可用：
- **法律检索**：关键词 `law`/`yuandian`/`pkulaw`/`法宝` → 映射 LAW.*
- **文档解析**：关键词 `parse`/`mineru`/`百炼`/`markdown`/`docling` → 映射 PARSE.*

**探测结果处理**：
- **法律检索可用** → 复盘报告中的法条和案例引用自动校验，静默继续。
- **法律检索不可用** → 用 `AskUserQuestion` 告知用户："法律检索 MCP 未连接，复盘报告中的法条和案例引用将无法自动校验。建议前往 QoderWork → 设置 → 连接器 中配置元典或北大法宝。是否暂不配置、以降级模式继续？"
  - 用户选择继续 → 降级执行，法条标注 `[L4-法条待验证]`，案例标注 `[L4-案例待验证]`
- **文档解析可用** → 上传裁判文书自动解析为 Markdown，静默继续。
- **文档解析不可用** → 按降级链回退（内置 pypdf → fitz → Read），或提示用户粘贴文书文本。

> **配置引导**：如需启用自动法条校验，请前往 QoderWork → 设置 → 连接器 配置元典或北大法宝。
> 探测协议与降级规则详见 qoder.md「MCP 预检协议」。

## 1 | 材料采集

- 裁判文书（必备）：经文档解析（解析后端，见 qoder.md「外部能力后端」）或用户粘贴。
- 自办深度还需：本案台账记录、事实梳理、时间轴、证据目录、质证意见、代理词、庭审笔录/复盘记录。
- 庭审复盘还需：庭审记录。三级降级——①正式庭审笔录 → ②用户自制记录或简短说明 → ③无则跳过对应分析并标注。

---

## 2 | 庭审复盘（庭后，五维）

《案件庭审复盘报告》模板字段：案件名称、开庭时间、主审法官、审理法院、原告、被告、第三人。

正文板块：
1. **庭后待完善工作**：还需提交的补充意见 / 需核实后回复法庭的问题 / 完成期限（涉事实问题宜由当事人签字盖章并取回执）。
2. **庭审效果分析（五维）**：① 案件事实描述（有利要件事实是否全面陈述、有无无法举证的要件事实）；② 举证质证（己方举证能否证明要件事实、对方关键证据及影响）；③ 法官问题指向（从法庭发问看倾向性）；④ 代理意见陈述（是否完整清晰、对方反驳哪些有效/无效）；⑤ 法官自由心证偏向（判断偏向哪方、己方支持可能性）。
3. **需讨论的其他事项**。
4. **总结**。

> 二审庭审复盘差异：着重双方对一审查明事实有无异议、相关证据能否证明主张；评估是否申请组成合议庭正式开庭。衔接 `/代理词`（庭后补充代理意见）。

---

## 3 | 裁判文书研析报告（六维）

依裁判文书分析报告模板，逐维拆解（保持中立客观）：

1. **当事人及诉讼参与人**：是否遗漏当事人、是否违法缺席判决、审判组织组成是否合法、应回避未回避、出庭人员是否合规。
2. **诉请与判决结果**：判决主文与诉请逐项比对（是否漏判/超裁）、是否剥夺辩论权利、受理有无管辖问题、是否需在二审增加独立诉请；统计数字/计算（利息违约金基数与起止）有无错误。
3. **请求权分析**：原告请求权基础所依据的法律规范 vs 法院针对诉请适用的法律规范，二者是否矛盾/有区别。
4. **基本事实是否查明**：一审认定事实 → 是否错误认定 → 是否遗漏认定；逐项比对查明事实与质证情况、庭审笔录是否一致；建立《查明事实异议整理》（具体到页/行/句、错在哪、漏了什么、有何证据可证）。
5. **"本院认为"（说理）**：逐个争议焦点，结论是否明确准确、引用法律与事实依据是否支持结论、是否对每项诉请均裁决并评价构成要件——**提炼当事人未达目标的准确原因（胜败因）**。
6. **裁判结果与裁判依据**：裁决理由引用的法律/司法解释名称·条款·内容，是否涵盖并支持全部裁决结果。
+ **其他核对**：当事人信息、数字/地点、审判人员是否为开庭人员、是否盖章、文书日期（尤其年月）。
+ **律师声明**：仅供决策参考、不构成承诺、保密、不得转递第三方。

> 自办深度：结合本案台账/质证/代理词，定位"我方主张为何被采纳/未采纳"，导出上诉或救济要点。
> 非自办降级：仅就文书文本分析事实认定与说理逻辑，结论标「基于受限信息，未掌握完整卷宗」。

---

## 4 | 结案复盘报告（结案报告 + 四维反思）

**结案报告十项固定字段**：委托人（律师）/ 所属业务线（法务）、对方当事人、主办人、案由、审理机关、审理结果、案号、案情简介、团队工作（按时间线逐条列明）、案件总结。

**案件总结五要点**：该案难点、法院对争议焦点如何认定、争议焦点与预判是否一致、最终结果是否符合委托前预期、承办人付出哪些工作。

**四维反思方法论**：① 案件存在的遗漏与不足（承认思维/策略/准备/知识漏洞 → 查清根源 + 找规避方法）；② 梳理事实证据法律关系、寻求类型化（提炼该类案件一般环节，提升同类办案水准）；③ 明晰法官思维、梳理论证理路；④ 案涉法律规范的研究探析。

> 自办深度：工作时间线自动从台账 `history.md` 生成；敞口校准引用台账"初始预判 vs 最终结果"。
> 法务附加项：外部律师表现评估（响应速度/专业质量/费用控制/沟通配合）→ 回写台账外部律师服务评价字段 → 作为下次委外选择参考。
> 二审结案报告：框架同上，按代理程序区分。

---

## 5 | 输出与汇报

1. **受众适配**（按 `qoder.md` 角色与内部汇报风格）：
   - **律师→客户版**：强调工作量与结果、消除疑惑、有始有终；律师工作底稿与内部反思不对外。
   - **律师→内部版**：含坦诚反思与改进计划。
   - **法务→领导/业务部门版**：结论先行 + 风险量化 + 经验教训 + 外部律师表现评价；如涉重大性/计提，附财务影响。
   - **法务→知识管理版**：类型化经验提炼，供同类案件参考。
2. **⛔ 输出前必过闸门**：报告引用的法规/法条/案例（含案号）在交付前先过 `/法律核验`，判定全为「准确」才放行；法律检索后端不可用标 `[L4-法条待验证]` 并提示人工复核，不跳过闸门。
3. 核心交付 `.md`；放行后询问用户是否经 `Word 转换后端`（DOCX.md_to_docx）（preset=legal）转 Word 供汇报。文末以 `file://` 回链。
4. **衔接**：结案复盘完成 → `/案件管家`（标结案/归档）、`/诉讼知识库`（把研析与复盘沉淀进知识库，类型化经验可复用）。

---

## 6 | 执行规则

1. **客观中立**：研析判决保持中立解读，不带情绪、不护短。
2. **不预测**：复盘聚焦已发生事实与经验，不预测其他案件结果。
3. **降级透明**：非自办/受限信息全程标注，不臆造办案过程。
4. **保密**：默认 CONFIDENTIAL；对外汇报版按受众脱敏。
5. **闸门不可绕过**：引用必过 `/法律核验`。
