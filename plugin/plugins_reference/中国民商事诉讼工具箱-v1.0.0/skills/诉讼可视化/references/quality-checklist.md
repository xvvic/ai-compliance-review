# 图表质量检查清单

生成或修订法律图表时的质量门控参考。

## 不合格输出（必须修正）

- 连线穿越非端点节点
- 连线标签与节点文字重叠
- 多条关系线挤在同一通道
- 节点中包含整句话而非精炼标签
- 一张图混合合同链、付款链、担保链而无视觉分离
- 节点重叠或间距过近
- 标签被遮挡或无白色背景
- 使用高饱和度颜色（大红大绿）
- 同类实体使用不同形状
- 同一实体在图中重复出现

## 修正规则

1. **添加枢纽节点**：多条箭头指向同一法律后果时，创建枢纽节点将详细项目连接到枢纽
2. **拆分图表**：超过6方或8条关系线时拆为专题图
3. **使用泳道**：法律主体上层/违约中层/索赔下层
4. **使用旁注**：视觉次要的法律关系放旁注或图例
5. **一线一义**：一条线只表达一种法律关系
6. **重排节点**：线条交叉时优先重排节点位置
7. **标签处理**：连线标签加白色背景，放在连线中部
8. **间距检查**：节点间最小间距40px，保留呼吸空间

## 构图四原则

| 原则 | 检查要点 |
|------|---------|
| 对比 | 不同元素是否有明显视觉区分？对比是否足够强烈？ |
| 重复 | 同类实体是否使用一致的形状、颜色、字号？ |
| 对齐 | 所有节点是否遵循网格或参考线对齐？ |
| 亲密性 | 相关元素是否紧邻？无关元素是否有足够分离？ |

## 3S 内容检查

| 维度 | 标准 |
|------|------|
| Simple | 节点标签2-4行、使用简称、不重复条文内容 |
| Straight | 事实表达直接、观点传达无断裂、逻辑链无缝 |
| Strategy | 内容服务于诉讼策略、保留支持己方的关键细节、去除无效信息 |

## 色彩检查

- 主色不超过4种
- 使用低饱和度色
- 同一实体全图同色
- 色彩有明确的法律语义
- 填充色和边框色配套
