# 庭审提纲

## 概述

本技能用于生成庭审提纲（含法庭询问清单）。整合为一个输出文件，包含两大板块：
- **板块一：庭审提纲** —— 我方应对预案（应对对方提问和法庭提问）
- **板块二：法庭询问清单** —— 我方主动询问策略（向对方发问的问题设计）

## 前置条件

### 主诉场景（我方为原告/申请人）

| 序号 | 材料 | 必要性 | 说明 |
|------|------|--------|------|
| 1 | 起诉状 | 必需 | 明确诉讼请求与事实理由 |
| 2 | 我方证据目录 | 必需 | 举证安排 |
| 3 | 我方证据册 | 必需 | 证据原文 |
| 4 | 要件攻防分析 | 必需 | 各请求权要件的攻防态势 |
| 5 | 对方答辩状（如有） | 建议 | 了解对方抗辩方向 |

### 被诉场景（我方为被告/被申请人）

| 序号 | 材料 | 必要性 | 说明 |
|------|------|--------|------|
| 1 | 答辩状 | 必需 | 我方抗辩体系 |
| 2 | 双方证据目录 | 必需 | 全面掌握举证情况 |
| 3 | 双方证据册 | 必需 | 证据原文 |
| 4 | 要件攻防分析 | 必需 | 各请求权要件的攻防态势 |
| 5 | 原告起诉状 | 必需 | 对方主张体系 |

如关键材料缺失，提示用户补充后再生成。

## MCP 预检

> 预检协议详见 qoder.md「MCP 预检协议」。本技能使用后端：LAW.*（法条储备和法条速查的自动填充与校验）。
> 不可用时降级标注：`[L4-法条待验证]`。

## 工作流

### Step 1: 立场确认与材料解析

1. 确认我方诉讼地位（原告/被告/第三人）
2. 解析全部前置材料，提取：
   - 双方诉讼请求/抗辩主张
   - 争议焦点清单
   - 各方证据体系概览
   - 要件攻防中的薄弱环节

### Step 2: 争议焦点梳理

1. 根据要件攻防分析，列出预判的争议焦点
2. 按重要性排序
3. 标注各焦点中我方的攻防态势（优势/劣势/僵持）

### Step 3: 生成板块一——庭审提纲

按庭审程序三阶段组织：

#### 3.1 程序性事项预案

审查是否存在以下程序性问题，如有则准备应对方案：
- 管辖权问题
- 回避申请
- 追加/变更当事人
- 诉讼请求变更/增加
- 反诉应对
- 证据交换相关程序事项

#### 3.2 法庭调查阶段

针对每个争议焦点，准备：

```
争议焦点[N]：[焦点描述]
─────────────────────────────
【我方陈述要点】
- 事实主张：...
- 证据引用：证据[编号]证明...
- 关键表述（建议口径）：...

【预判对方可能提问】
Q1: [对方可能提出的问题]
  → 建议回答：...
  → 注意事项：...（避免自认/不要超出范围）

Q2: [对方可能提出的问题]
  → 建议回答：...

【预判法庭可能提问】
Q1: [法官可能关注的问题]
  → 建议回答：...
  → 补充说明：...

Q2: [法官可能关注的问题]
  → 建议回答：...

【底线/红线提醒】
- 绝不可认可的事实：...
- 可以适度让步的空间：...
```

#### 3.3 法庭辩论阶段

```
辩论要点[N]：[对应争议焦点]
─────────────────────────────
【我方辩论提纲】
- 第一层论点：...
  法律依据：[法条]
  事实依据：证据[编号]
- 第二层论点：...

【预判对方辩论方向】
- 对方可能主张：...
- 我方反驳策略：...

【法条储备】
- [相关法条及条文要点]
```

#### 3.4 最后陈述阶段

- 最后陈述要点提纲（精练版，3-5句话）
- 强调核心诉求/抗辩
- 情理结合的表达建议

### Step 4: 生成板块二——法庭询问清单

> 📎 加载 `references/examination-techniques.md` 作为法庭询问技巧与问题设计指南。

按争议焦点组织我方向对方的发问：

#### 询问清单设计原则

1. **目的性**：每个问题必须服务于特定待证事实
2. **层次性**：从基础事实到核心争议，逐步深入
3. **追问设计**：预判对方回答，准备追问方向
4. **非诱导性**：避免诱导性提问（法官可能制止）
5. **证据锚定**：问题设计结合已有证据，形成"证据+陈述"的印证

#### 询问清单格式

```
争议焦点[N]：[焦点描述]
待证事实：[通过询问拟证明的事实]
─────────────────────────────

问题1：[提问内容]
  目的：证明[待证事实]
  预判回答A：[对方可能回答] → 追问：[追问内容]
  预判回答B：[对方可能回答] → 追问：[追问内容]
  预判回答C：对方拒绝回答/回避 → 应对：[向法庭说明]

问题2：[提问内容]
  目的：...
  预判回答A：...
  ...

【备选问题】（视庭审情况决定是否使用）
- [备选问题1]：适用场景...
- [备选问题2]：适用场景...
```

### Step 5: 法条引用校验

> 法律检索后端探测与降级详见 qoder.md「MCP 预检协议」。

```
能力调用：LAW.article_detail / LAW.statute_detail（法条/法规校验）
检索范围：
- 实体法条文（对应案由相关）
- 程序法条文（举证/质证/辩论规则）
- 司法解释相关条款
```

> ⚠️ 如法律检索后端均不可用，法条引用标注为"[L4-法条待验证]"，提示用户自行核实。

### Step 6: 输出文档（md 优先）

> **⛔ 交付前必过闸门**（见 qoder.md「法律核验闸门」）：核验对象为提纲中法条引用。

**默认交付 Markdown**：核验放行后先产出 md 成品（写入输出目录并回链），随后询问用户「是否转 Word（Word 转换后端（DOCX.md_to_docx, preset=legal））」，确认才转 .docx。

> **输出文件命名**：`{案件简称}_庭审提纲_{日期}.md`。Word 版同名 `.docx`。

## 庭审应对策略指引

### 法庭调查阶段策略

1. **陈述要精炼**：法庭调查阶段以事实为主，法律论证留给辩论阶段
2. **举证要有序**：按证据目录顺序举证，关联证据一并出示
3. **回答要谨慎**：对方和法庭提问时，不确定的事实回答"需要庭后核实"
4. **记录要及时**：标注对方陈述中的矛盾点，为辩论和询问做准备

### 法庭辩论阶段策略

1. **结构化表达**：先总后分，让法官容易跟随
2. **法条先行**：每个论点先引法条，再结合事实
3. **回应+反击**：先回应对方辩论观点，再展开我方主张
4. **时间控制**：核心观点前置，次要观点视时间决定是否展开

### 询问技巧

1. **封闭式优先**：尽量使用封闭式问题，控制对方回答方向
2. **逐步推进**：从对方容易承认的事实出发，逐步引向核心事实
3. **善用沉默**：对方回避时，适当沉默后再次追问
4. **及时收束**：达到目的后立即停止，不给对方辩解机会

## 特别提醒

1. **保密义务**：提纲内容属于律师工作成果，提醒当事人不得泄露
2. **灵活应变**：提纲是预案而非剧本，庭审中需根据实际情况调整
3. **配合分工**：如有多位代理人，标注分工安排
4. **庭审礼仪**：提醒当事人庭审注意事项（着装、态度、发言规范）
5. **录音安排**：建议庭审全程录音（合法范围内），便于庭后复盘

## 下一步建议

庭审提纲完成后，根据案件进展可继续：
- **庭前代理词** → `/代理词`（模式 A）系统阐述己方主张
- **庭后补充** → `/代理词`（模式 B）庭审结束后根据庭审情况补充代理意见
- **法条储备** → `/法律核验` 复核提纲中法条储备板块的准确性

## 错误处理

- 要件攻防分析缺失 → 无法生成有针对性的提纲，要求用户先完成
- 对方主张不明确 → 基于起诉状/答辩状推断，标注"预判"
- 后端服务不可用 → 按 qoder.md 降级规则标注，继续完成提纲主体。
- 案件类型特殊（如知产/海商） → 提示可能需要补充专门程序规则
