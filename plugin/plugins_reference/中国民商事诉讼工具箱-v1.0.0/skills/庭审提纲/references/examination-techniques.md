# 法庭询问技巧指南

## 一、法庭询问的目的

1. **查明事实**：澄清案件关键事实，填补证据空白
2. **固定证据**：引导对方确认对我方有利的事实
3. **暴露矛盾**：揭示对方陈述与证据之间的矛盾
4. **动摇对方**：通过追问削弱对方核心主张的可信度
5. **铺垫论证**：为后续法庭辩论中的法律论证奠定事实基础

## 二、问题设计核心原则

### 2.1 针对性原则
每个问题必须服务于特定的证明目的，与案件争点直接相关。

### 2.2 封闭性原则
优先设计封闭式问题（是/否、确认具体事实），避免开放式问题。

| 问题类型 | 特点 | 适用场景 |
|---------|------|---------|
| 封闭式 | 限定回答范围 | 确认事实、固定证据 |
| 开放式 | 对方自由回答 | 需要对方解释说明（慎用） |

### 2.3 递进性原则
问题之间形成逻辑链条，从基础事实到争议焦点，层层深入。

```
第一层：基础事实确认
    ↓
第二层：具体细节追问
    ↓
第三层：矛盾点揭示
    ↓
第四层：不利推论迫使
```

### 2.4 可验证性原则
问题涉及的事实应有证据支撑或可验证。

## 三、问题分类

### 3.1 基础确认类
目的：确认无争议事实，建立询问基础。设计要点：选择对方难以否认的事实，为后续问题做铺垫。

### 3.2 细节追问类
目的：澄清关键细节，填补证据空白。设计要点：针对证据中的模糊之处，追问具体时间、地点、金额。

### 3.3 矛盾揭示类
目的：暴露对方陈述与证据之间的矛盾。设计要点：先确认对方陈述→展示矛盾证据→迫使对方解释。

### 3.4 逻辑推演类
目的：通过逻辑推演，迫使对方承认不利推论。设计要点：让对方在每个节点难以否认，最终得出对我方有利的结论。

### 3.5 证据质疑类
目的：质疑对方证据的真实性、合法性或关联性。设计要点：询问证据的形成过程，揭示瑕疵。

## 四、追问设计技巧

### 4.1 分支式追问
对每个核心问题设计多种可能的回答路径及对应追问：
```
主问题：[核心问题]
├─ 路径A：回答"是/确认"
│   ├─ 追问A1：[固定有利事实]
│   └─ 追问A2：[深化追问]
└─ 路径B：回答"否/否认"
    ├─ 追问B1：[展示矛盾证据]
    └─ 追问B2：[迫使承认]
```

### 4.2 陷阱式追问
设计看似无害但暗藏陷阱的问题。例："除了您刚才提到的质量问题，还有其他拒付货款的理由吗？"

### 4.3 确认式追问
对关键事实反复确认，形成"锁定"效果：逐步确认前提→得出结论。

## 五、询问策略

### 5.1 主诉案件（原告）
目标：证明请求权基础构成要件成立。重点方向：合同成立与生效→己方履行→对方违约→损失。

### 5.2 被诉案件（被告）
目标：动摇对方请求权基础，证明抗辩事由成立。重点方向：合同效力瑕疵→己方已履行→对方违约在先→损失计算错误。

## 六、问题优先级

| 优先级 | 类型 | 说明 |
|-------|------|------|
| P0 | 核心事实确认 | 涉及请求权基础构成要件 |
| P1 | 重要细节追问 | 填补证据空白、强化论证 |
| P2 | 辅助性问题 | 增强说服力但非必需 |
| P3 | 备用应变问题 | 根据庭审情况决定 |

## 七、常见错误与禁忌

### 错误
1. 问题过长，对方难以理解
2. 问题模糊，对方可以随意发挥
3. 诱导性过强，被法官制止
4. 未准备追问，对方回答后无法继续

### 禁忌
- 暴露我方弱点的问题
- 引发新争议的问题
- 无法证实的主观意图问题
- 攻击性过强引起反感的问题

## 八、庭审应变

### 对方拒绝回答
1. 申请法庭责令回答
2. 说明不利推定后果
3. 转入备用问题

### 对方回答模糊
1. 要求明确"是"或"否"
2. 细化问题逐一询问
3. 结合证据追问

### 对方提出新事实
1. 核实新事实的证据支撑
2. 申请补充举证期限
3. 确保记录在案
