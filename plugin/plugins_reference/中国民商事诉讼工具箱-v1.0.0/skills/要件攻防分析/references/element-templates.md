# 常见请求权审查框架模板

> 覆盖民事诉讼中最常见请求权类型的审查框架起始模板。为要件攻防分析提供快速构建骨架。

## 凡例

- 阿拉伯数字指向《民法典》条-款，如"509-I"=第509条第1款
- "Nr."=项，如"563Nr.4"=第563条第(四)项

---

## 一、合同原给付请求权

```
请求权已产生
├── 合同成立
│   ├── 要约：472（构成）+ 137/138/139（生效）
│   ├── 承诺：479（构成）+ 488/489（一致性）
│   └── 特别要件：490（要式）/ 496-II（格式条款订入）
├── 合同生效
│   ├── 行为能力：144（无行为能力无效）/ 145-I（限制行为能力）
│   ├── 意思表示无瑕疵：146-151（可撤销事由）
│   ├── 内容合法：153-I / 153-II / 154
│   ├── 生效条件：158（附条件）/ 160（附期限）/ 502-II（审批）
│   └── 代理权：162 / 171 / 172
├── 履行期限届至
└── 具体给付义务内容

请求权未消灭
├── 557-I（债的消灭事由）
├── 解除：562/563 + 565 + 566
└── 撤销权行使：152

请求权可行使
├── 525（同时履行抗辩）
├── 526（先履行抗辩）
├── 527（不安抗辩）
└── 192-I（诉讼时效抗辩）
```

## 二、违约损害赔偿请求权

```
请求权已产生
├── 责任成立
│   ├── 合同有效成立（同上）
│   ├── 义务违反
│   │   ├── 义务类型：主给付/从给付/附随义务(509-II)
│   │   ├── 违反形态：不能/迟延/不完全履行/加害给付
│   │   └── 第580条-I但书（履行不能排除）
│   └── 可归责性
│       └── 不可抗力抗辩：590（防御规范）
├── 责任范围
│   ├── 损害：584主文（实际损失+可得利益）
│   └── 可预见性限制：584但书

请求权未消灭
├── 557-I
├── 591-I（减损义务违反→扣减）
└── 损益相抵

请求权可行使
├── 525-527（履行抗辩权）
└── 192-I（诉讼时效）
```

## 五、过错侵权损害赔偿请求权

请求权基础：1165-I

### 审查框架一：作为侵权 + 绝对权

```
请求权已产生
├── 责任成立
│   ├── 事实构成 [原告举证]
│   │   ├── 适格权益被侵：109-114
│   │   ├── 加害行为（作为）
│   │   └── 责任成立因果关系
│   │       ├── 否认：受害人故意 1174
│   │       └── 否认：第三人原因 1175
│   ├── 抗辩：不法性阻却 [被告举证]
│   │   ├── 正当防卫：181
│   │   ├── 紧急避险：182
│   │   ├── 自助行为：1177
│   │   └── 自甘冒险：1176
│   ├── 过错（故意/过失）[原告举证]
│   └── 抗辩：与有过失（损害发生）：1173
├── 责任范围
│   ├── 损害 [原告举证]
│   │   ├── 人身损害：1179
│   │   ├── 财产损害：1184
│   │   └── 精神损害：1183
│   ├── 责任范围因果关系
│   └── 抗辩：与有过失（损害扩大）：1173

请求权未消灭：557-I / 损益相抵
请求权可行使：192-I（三年，188起算）
```

### 审查框架二：不作为侵权 + 绝对权

```
请求权已产生
├── 适格权益被侵
├── 作为义务 [原告举证]
│   └── 来源：法律/合同/先行行为/安全保障义务(1198)
├── 否认事由 [被告反证]
│   ├── 已尽作为义务
│   ├── 无避免可能性
│   └── 受害人故意/第三人原因
├── 责任范围（同上）
```

### 审查框架三：纯粹经济损失

```
请求权已产生
├── 纯粹经济损失
├── 行为不法性 [原告积极证明]
│   ├── 路径①：违反保护性法律
│   └── 路径②：故意悖俗
├── 因果关系 + 可归责性
├── 责任范围（同上）
```

```
└── 无法律上原因

├── 抗辩：985但书

请求权未消灭：557-I / 986善意得利人限制
请求权可行使：192-I
```

## 使用说明

1. 本文件为快速起步模板，非完整知识库。特殊情形须通过法条检索工具自主构建。
2. 骨架中每个条文号在正式使用前须经实时法条检索确认现行有效性。
3. 任何未覆盖的请求权类型，均可按 `methodology/outline-engine.md` 的流程从零构建。
