# 攻防框架构建

> 从案件事实出发，经"找法→分类→构建→拆解→成型"五步，产出完整的攻防审查框架。

## 一、五步流程

```
Step 1: 找法
  案件事实 → 识别法律关系 → 确定诉请类型 → 按优先级排查请求权基础 → MCP 锁定条文
  优先级：合同 → 类合同 → 无因管理 → 物权 → 侵权 → 不当得利（兜底）

Step 2: 规范分类
  对每个候选条文判定性质：
  - 请求权基础：赋予一方请求权 → 原告举证全部要件
  - 辅助规范：补充某个要件的含义
  - 防御规范：阻却/消灭/阻止请求权 → 被告举证
  关键区分：否认（否定对方事实）vs 抗辩（引入新事实）

Step 3: 构建审查框架
  以请求权基础为核心，按三层结构展开：
  第一层（权利产生）：成立要件 + 产生抗辩
  第二层（权利未消灭）：消灭抗辩（清偿/解除/撤销…）
  第三层（权利可行使）：行使抗辩（时效/同时履行/不安…）
  → 参照 references/element-templates.md 快速起步

Step 4: 要件拆解
  将每个构成要件拆解为最小审查单元：
  - 按"且/或"拆分条文条件
  - 识别隐含要件（因果关系、过错、行为能力）
  - 标注不确定法律概念（需解释→触发 legal-interpretation-toolkit.md）

Step 5: 攻防框架成型
  组装：审查框架 + 事实映射 + 争点标记
  争点等级：◎核心争点 / ○次要争议 / ·明显满足
  → AskUserQuestion 确认后进入正式论证
```

## 二、找法常见遗漏

1. 遗漏竞合请求权（违约与侵权竞合，第186条）
2. 忽视第三人请求权（合同相对性例外，如第522条）
3. 混淆诉讼类型（解除=形成之诉；解除后返还=给付之诉）
4. 遗漏附随义务（第509条第2款）产生的独立请求权
5. 忽视特别法中的请求权基础

## 三、否认与抗辩的判别

核心区别：被告是在说"你说的不是真的"（否认）还是"即使你说的是真的，但有其他事实导致你不能赢"（抗辩）？

| 维度 | 否认 | 抗辩 |
|------|------|------|
| 事实 | 否定对方主张事实 | 承认对方事实+引入新事实 |
| 举证 | 反证（动摇心证即可） | 本证（使法官确信） |
| 位置 | 构成要件项下 | 独立抗辩项下 |

实战意义：归位错误会导致举证责任分配错误、攻防结构图节点位置错误。

## 四、MCP 工具依赖

| 能力 | 用途 |
|------|------|
| `LAW.statute_search` | 检索法规名称 |
| `LAW.article_detail` | 获取条文原文 + 现行有效性 |
| `LAW.article_search` | 按关键词检索条文 |
| `LAW.semantic_search` | 语义检索相关法条 |

降级：MCP 不可用时基于方法论知识独立构建，标注 `[方法论自主构建]`。

## 五、攻防框架输出格式

```
A→B / 诉请1 / §577（违约赔偿）
  产生：合同有效成立(·) → 义务违反(○) → 归责性(·) → 损害(◎) → 因果(○)
  消灭：清偿(·否) / 减损义务(○)
  行使：同时履行(·否) / 时效(·否)
  结论：成立，取决于损害范围
```
