# 法庭报告技术

> 本文件描述要件攻防分析的核心工作方法——如何在原被告各执一词时，按阶段审查各方主张的法律合理性。

## 一、为什么要分阶段审查

实际诉讼中原被告各说各话。攻防分析不试图一次性判断"谁对谁错"，而是**分阶段、分视角**地审查各方主张——先看原告的故事在法律上是否站得住，再看被告的防御是否站得住。经法律审查筛选后，大量争议点不必进入证据比拼即可得出方向。

## 二、审理原则

```
程序先于实体 → 法律先于事实 → 原告先于被告
```

- **程序先于实体**：先看管辖/主体/期限等程序问题
- **法律先于事实**：先做法律合理性审查（各方陈述能否对应规范），再看证据
- **原告先于被告**：先审原告主张，确认站得住后才审被告防御

## 三、五阶段流程

### 第一阶段：程序审查

诉讼程序要件（管辖/主体/时效/格式等）。本 skill 中此步交由 `/程序性文书系列` 处理。

### 第二阶段：原告阶段

**核心问题**：假设原告说的都是真的，原告的诉请在法律上能否成立？

- 审查对象：仅原告的主张和陈述
- 审查标准：陈述能否满足请求权基础的各项构成要件
- 不涉及证据——不问"原告能否证明"，只问"原告说的这些如果为真，法律上能不能赢"
- 原告陈述中对己不利的内容也须审查
- 不替原告选择请求权基础——遍检所有候选

**三种结论**：

| 结论 | 后果 |
|------|------|
| 法律上站不住 | 终结——原告败诉方向明确 |
| 部分站得住 | 仅合理部分进入被告阶段 |
| 法律上站得住 | 进入被告阶段 |

### 第三阶段：被告阶段

**前提**：原告主张至少部分在法律上站得住。

**核心问题**：假设被告说的都是真的，被告的防御在法律上能否成立？

- 审查对象：仅被告的抗辩和陈述
- 审查标准：陈述能否满足抗辩规范的各项要件

**被告防御手段**：

| 类型 | 性质 | 举证后果 |
|------|------|---------|
| 否认 | 否认原告陈述的事实 | 无须举证新事实 |
| 权利障碍抗辩 | 主张权利从未产生 | 被告举证 |
| 权利消灭抗辩 | 主张权利已消灭（清偿/免除/抵销…） | 被告举证 |
| 权利阻止抗辩 | 主张暂不可行使（时效/同时履行…） | 被告举证 |

**关键规则**：
- 只要一项请求权基础未被合理防御 → 原告因该项胜诉
- 所有请求权基础均被合理防御 → 被告防御整体成立

**三种结论**：

| 结论 | 后果 |
|------|------|
| 防御站不住 | 终结——原告不经举证胜诉 |
| 部分站得住 | 未被防御的请求权基础直接成立 |
| 防御站得住 | 进入反抗辩/证据阶段 |

### 反抗辩与再抗辩

- **反抗辩**：被告抗辩站得住时，原告可提反击（如"解除无效"回应"已解除"）
- **再抗辩**：原告反抗辩站得住时，被告可再反击
- 审查方式同上——一方无合理回应即对方胜诉

### 第四阶段：证据阶段

**前提**：双方主张都在法律上站得住，且存在事实矛盾。

**规则**：
- 一方主张 + 对方认可 → 无需举证
- 双方有争议 + 该事实具有法律意义 → 列入待证事实清单
- 举证责任：成立要件→原告；消灭/阻止→被告；法定倒置从其规定

本 skill 产出待证事实清单与举证责任分配，后续交付 `/证据目录`、`/质证意见`、`/庭审提纲`。

## 四、效率原理

1. 经法律审查后，大量案件不必进入证据比拼即可结案
2. 仅有争议且有法律意义的事项才需举证——避免多余举证
3. 审查范围在各阶段逐步限缩——既无遗漏也无冗余

## 五、速查表

| 阶段 | 审查问题 | 对象 | 结论 |
|------|---------|------|------|
| 程序 | 起诉合规？ | — | 合格/不合格 |
| 原告 | 原告主张法律上站得住？ | 仅原告陈述 | 站得住/站不住 |
| 被告 | 被告防御法律上站得住？ | 仅被告陈述 | 站得住/站不住 |
| 反抗辩 | 原告反击站得住？ | 第二个原告阶段 | 站得住/站不住 |
| 再抗辩 | 被告再反击站得住？ | 第二个被告阶段 | 站得住/站不住 |
| 证据 | 争议事实由谁举证？ | 仅争议+有法律意义事项 | 举证分配 |
