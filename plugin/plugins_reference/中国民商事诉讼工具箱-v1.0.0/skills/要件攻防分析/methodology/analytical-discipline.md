# 分析纪律

> 攻防分析过程中的法条引用铁律、术语规范与品质约束。

## 一、法条引用铁律（MCP 校验）

1. **首次必校验**：任何条文号/原文首次引用前，须调用法律检索后端 `LAW.article_detail` 获取原文 + 现行有效性。
2. **后续可复用**：同一条文在同一分析中再次引用不必重复调用。
3. **以 MCP 为准**：MCP 返回与其他来源不一致时以 MCP 为准。
4. **禁用模型记忆**：基于训练记忆直接落笔法条，严令禁止。
5. **降级标注**：MCP 不可用时，全部法条引用带 `[L4-法条待验证]`，流程不阻断。

**引用格式**：首次引用展开完整原文（引用块），后续用简称+条文号。

## 二、术语统一

| 术语 | 适用场景 |
|------|---------|
| 请求权基础 | 给付之诉 |
| 规范基础 | 确认之诉/形成之诉/非讼程序 |
| 权利已产生/未消灭/可行使 | 三层审查 |
| 否认 | 否定对方要件事实的真实性（无主张责任） |
| 抗辩 | 引入新事实阻却/消灭/阻止请求权（有主张责任） |

## 三、来源标注

| 来源类别 | 标注方式 |
|---------|---------|
| 经 MCP 校验的法条 | 条文号 + 校验标记 |
| 方法论自主分析 | 无脚注，正文显性展开推理 |
| 模型推断 | 附加 `[待核验]` 或触发 MCP 校验 |

**禁止**：将模型记忆包装为"通说认为"；省略来源类别；使用"显然""不言自明"。

## 四、标记词汇（五种）

| 标记 | 含义 |
|------|------|
| `[待验证:事实]` | 事实主张待证据验证 |
| `[需引用:法条]` | 法律依据待 MCP 校验补充 |
| `[请审阅:主观判断]` | 含分析者主观评价 |
| `[证据缺口]` | 举证不足 |
| `[风险提示]` | 对己方不利因素 |

## 五、论证品质底线

- 争议要件须完成"规范含义→事实对照→结论"的完整论证。
- 否认与抗辩严格区分、归位正确。
- 核心争议深入论证，明显要件直接论断（参 `proportionality.md`）。
- 不确定概念经界定后方可适用（参 `legal-interpretation-toolkit.md`）。
- 客观第三人称，避免绝对化表达。
