# 请求权检索顺序

> 当案件涉及多个当事人和多项请求权时，按何种顺序排列审查。

## 一、五级定位

### 第一级：当事人关系（"谁向谁"）

列出全部当事人，两两建立请求权可能性矩阵（A→B、B→A、A→C…），每对关系独立审查。

### 第二级：诉讼请求（"主张什么"）

对每对关系识别具体诉请类型：

- 给付之诉：要求作为/不作为/支付金钱/移转所有权等
- 确认之诉：确认法律关系存在或不存在
- 形成之诉：变更或消灭法律关系（撤销、解除）
- 非讼程序：实现担保物权、人身保护令等

每项诉请独立成章。

### 第三级：规范类别（"基于什么类型的权利"）

按以下类别预选候选请求权基础：

| 序号 | 类别 | 典型范例 |
|------|------|---------|
| 1 | 合同请求权 | 买卖/租赁/承揽/运输等给付义务 |
| 2 | 准合同请求权 | 无因管理、不当得利 |
| 3 | 物权请求权 | 返还原物/排除妨害/消除危险 |
| 4 | 占有保护请求权 | 占有返还/排除妨害/损害赔偿 |
| 5 | 人格权请求权 | 停止侵害/消除影响/赔礼道歉 |
| 6 | 知识产权请求权 | 停止侵害/销毁/赔偿损失 |
| 7 | 侵权损害赔偿 | 一般侵权/特殊侵权（产品/医疗/环境/交通） |
| 8 | 缔约过失赔偿 | 违反先合同义务 |
| 9 | 后合同义务 | 违反后合同义务的赔偿 |
| 10 | 婚姻家庭 | 抚养费/扶养费/赡养费/探望权 |
| 11 | 继承 | 法定继承/遗嘱继承/遗产管理 |
| 12 | 债的保全 | 代位权/撤销权 |
| 13 | 债权债务转移 | 债权转让/债务承担/债务加入 |
| 14 | 担保物权 | 抵押/质押/留置 |
| 15 | 确认之诉规范基础 | 合同效力/所有权/物权确认 |
| 16 | 形成之诉规范基础 | 合同撤销/解除/婚姻撤销 |
| 17 | 非讼程序规范基础 | 实现担保物权/人身保护令 |
| 18 | 程序性请求 | 管辖/保全/回避 |

### 第四级：具体规范（"基于哪一条文"）

在第三级类别中定位具体的请求权基础（精确到条、款、项）。通过攻防框架构建流程（`outline-engine.md`）锁定条文并构建审查框架；如用户已配置请求权基础知识库，优先参考。

### 第五级：逐项审查（"按什么步骤"）

对每项具体请求权基础，依三层审查结构（`inner-structure.md`）展开实质审查。这一级是分析主体。

## 二、排序优先级

多个请求权时按以下顺序排列：

1. **当事人**：原告→被告→第三人
2. **权利类型**：合同→准合同→物权→占有→人格权→侵权→不当得利（兜底）
   - 依据：特殊优先于一般——合同关系是当事人自愿安排，应首先穷尽
3. **聚合与竞合**：
   - 聚合的请求权分别独立审查
   - 竞合的请求权分别独立审查，结论部分说明可择一行使

## 三、与三层审查的对接

本文件决定"审查哪几项请求权、按什么顺序排列"；`inner-structure.md` 决定"每项请求权内部怎么审查"。五级定位完成后，以结果作为候选清单提交用户确认，确认后进入攻防框架构建和正式论证。
