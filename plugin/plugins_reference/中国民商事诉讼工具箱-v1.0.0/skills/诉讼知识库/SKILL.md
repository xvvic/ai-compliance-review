# 诉讼知识库 · 知识沉淀与问答

> 让每一次法律研究、新法解读、裁判文书研析与结案复盘，不止于一份文档，而是汇入不断生长的知识体系。本技能以 **三板块架构**（办案笔记/工作笔记/法律知识库）组织知识资产，以 qmind 语义检索为核心能力，以协作平台文档（`COLLAB.doc_*`，钉钉/飞书/同类）为可选团队协作载体。

---

## 0 | 前置

1. 读 `qoder.md`「知识库方案」：检查三个板块的 notebook ID 是否已配置。
2. **MCP 预检**：调用 `qw_mcp_list` 探测知识库后端是否可用：
   - qmind：关键词匹配 `qmind`/`knowledge` → 映射 qmind 语义检索能力
   - 协作平台：关键词匹配 `dws`/`dingtalk`/`钉钉`/`lark`/`feishu`/`飞书` → 映射 `COLLAB.doc_*` 能力
   - **qmind 可用** → 首选 qmind 云端知识库，静默继续。
   - **qmind 不可用、协作平台可用** → 降级到协作平台文档模式，静默继续。
   - **均不可用** → 用 `AskUserQuestion` 告知用户："知识库载体（qmind/协作平台文档）均未配置。是否先配置？配置后可实现知识持久化和团队共享。"
     - 用户选择配置 → 引导进入第 5 节知识库初始化引导
     - 用户选择继续 → 降级执行，标注 `[L4-知识库不可用]`
3. **未配置时**：引导用户完成知识库初始化（见第 5 节），或提示"请先运行 /冷启动访谈 配置知识库"。
4. 判定意图：**沉淀**（把成果归入）/ **问答**（向知识库提问）/ **状态查看**（检查各板块状态）。

## 1 | 三板块架构

| 板块 | 定位 | 典型内容 | 沉淀来源 |
|------|------|---------|---------|
| **办案笔记** | 按案件维度的实战经验 | 庭审反思、攻防策略、法官倾向、类案对比、结案教训 | /案件复盘、/要件攻防分析、用户上传 |
| **工作笔记** | 按主题维度的研究积累 | 法律研究、新法解读、方法论、行业观察、政策动态 | /法律研究报告、/新法解读、用户上传 |
| **法律知识库** | 法律规范与学理资料 | 法律评注、司法解释理解、请求权基础、论文摘要 | 用户上传（评注/讲座/论文等） |

**板块独立性**：每个板块对应一个独立的 qmind notebook（或协作平台文档空间），互不混淆。用户可按需只配其中一两个板块。

---

## 2 | 沉淀流程（主用例）

> 触发场景：`/法律研究报告`、`/新法解读`、`/案件复盘` 产出后，中枢推荐"沉淀到知识库"。

1. **确定目标板块**：根据产出类型自动路由（见 qoder.md「知识库沉淀路由」），用户可覆盖。
2. **生成沉淀摘要**：为成果生成结构化摘要（≤500字），含：标题、日期、核心发现、关键词标签、来源案件/法规。
3. **写入知识库**：
   - **qmind 模式**：将成果 md（含摘要头）上传到对应板块的 notebook，由 qmind 自动索引。
   - **协作文档模式**：经 `COLLAB.doc_create` 写入对应知识空间，按分类目录归档。
4. **确认落地**：回读确认写入成功，在对话中告知用户沉淀完成。

**沉淀格式规范**：
```markdown
---
title: {成果标题}
date: {YYYY-MM-DD}
source_skill: {产出技能名}
case_ref: {关联案件编号，如有}
tags: [关键词1, 关键词2, ...]
---

{成果正文或核心摘要}
```

## 3 | 知识库问答

> 触发场景：用户问"知识库里有没有类似的……""之前研究过 XX 问题吗""找一下之前的裁判规则"。

**问答流程**：
1. 判定应检索哪个板块（办案笔记/工作笔记/法律知识库），不确定时跨板块检索。
2. 调用 qmind retrieve 语义检索（或协作平台文档关键字搜索 `COLLAB.doc_read`）。
3. 基于检索结果组织回答，标注来源。
4. 如有相关发现，引导用户深入阅读原文。

**回答标注**：`[KB:办案笔记]`、`[KB:工作笔记]`、`[KB:法律知识库]` 标注知识来源。

**跨板块检索**：用户问题可能跨板块时，依次检索所有已配置板块，合并结果呈现。

## 4 | 两种载体方案

### 方案一：qmind 云端知识库（推荐）

- **优势**：语义检索、自动索引、RAG 问答、来源引用
- **配置**：在 qmind 平台创建 notebook → 获取 notebook ID → 写入 qoder.md
- **调用**：通过 qmind CLI（retrieve/rag 命令）

### 方案二：协作平台文档（COLLAB.doc_*，团队协作）

- **优势**：团队共享、文档协作、权限管理
- **配置**：提供文档空间链接（钉钉知识库 / 飞书文档空间）→ 提取 workspaceId → 写入 qoder.md
- **调用**：经 `COLLAB.doc_create` / `COLLAB.doc_read`（运行时映射到 dws 或 lark-cli）

### 降级策略

- qmind 不可用 → 降级到协作平台文档（如已配置 `COLLAB.doc_*`）
- 协作平台也不可用 → 用 `AskUserQuestion` 告知用户："知识库载体（qmind/协作平台文档）均未配置，知识将无法持久化，仅在对话内提供回答。是否先配置？配置后可实现知识持久化和团队共享。"
  - 用户选择配置 → 引导进入第 5 节知识库初始化引导
  - 用户选择继续 → 仅在对话内提供知识（不持久化），标注 `[L4-知识库不可用]`
- **不使用本地文件作为知识库**（纯净版不含本地知识库方案）

> **配置引导**：如需启用知识持久化，请前往 QoderWork → 设置 → 连接器 配置钉钉文档或安装 qmind skill。

## 5 | 知识库初始化引导

> 当用户首次使用本技能且 qoder.md 中知识库未配置时执行。

### qmind 方案引导

1. 确认用户已安装 `qmind-knowledge` skill（提供 qmind CLI）。
2. 引导用户在 qmind 平台创建三个 notebook：
   - 「办案笔记」—— 按案件分类的实战经验
   - 「工作笔记」—— 按主题分类的研究积累
   - 「法律知识库」—— 法律规范与学理资料
3. 获取各 notebook ID，写回 qoder.md「知识库方案 → qmind 注册表」。
4. 验证连通性（试跑一次 retrieve）。

### 协作平台文档方案引导

1. 引导用户提供文档空间链接（钉钉：`https://alidocs.dingtalk.com/i/spaces/{workspaceId}/overview`；飞书：对应空间链接）。
2. 确认对应空间下建立三个分类文件夹（办案笔记/工作笔记/法律知识库）。
3. 写回 qoder.md 对应配置。

> 两种方案可并行配置（qmind 做语义检索主力，协作文档做团队协作载体），也可只选其一。

## 6 | 跨技能衔接

| 触发 | 衔接 |
|------|------|
| /法律研究报告 产出 | → 沉淀到「工作笔记」 |
| /新法解读 产出 | → 沉淀到「工作笔记」 |
| /案件复盘 产出 | → 沉淀到「办案笔记」 |
| /要件攻防分析 启动 | → 先检索「办案笔记」寻找类似案件经验 |
| /法律研究报告 启动 | → 先检索「工作笔记」避免重复研究 |
| 用户上传法律资料 | → 归入「法律知识库」 |

---

## 7 | 执行规则

1. **只增不改**：沉淀内容只新增，不修改已入库的历史文档。
2. **不臆造 ID**：notebook ID / workspaceId 全部来自用户配置或实际命令返回。
3. **保密分级**：涉案内容遵守保密层级，团队知识库写入遵守权限。
4. **凭证安全**：qmind token 等凭证不输出到文档中。
5. **板块边界清晰**：办案笔记（案件维度）、工作笔记（主题维度）、法律知识库（规范维度）不混淆。
