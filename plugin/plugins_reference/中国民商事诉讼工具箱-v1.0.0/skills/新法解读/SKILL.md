# 新法解读

把刚发布或修订的法律法规，从"法条复读"升级为"实务映射"。本技能对标头部律所数百篇新法解读的共性手法——**以新旧对比表为基石，做制度提炼与风险量化，落到用户所处行业/客户领域的合规动作**，输出一份结构清晰、依据透明的《新法解读》（**仅 Markdown 为核心交付**，按应用目标可选导出公众号 HTML 或 Word）。

与 `/法律研究报告` 的分工：法律研究报告回答"某个具体法律问题该怎么定性/有何风险"；本技能针对"一部新发布或修订的法规整体"做体系化解读，核心差异是 **新旧对比驱动** 与 **行业落地清单**。

## 核心设计

### A. 双视角结论导向（读画像自动切换）

| 视角 | 适用画像 | 解读落点 | 关键措辞 |
|------|----------|----------|----------|
| **律师视角** | 外部律师 / 执业律师 | 新规对**所服务客户/执业领域**的影响 + 律师可提供的服务切入点 | "客户/委托人""可协助客户……""业务机会与风险提示" |
| **法务视角** | 公司法务 / 法律顾问 / 合规 | 新规对**本公司业务形态/所处行业**的影响 + 法务的内部合规动作 | "我司/本公司""内部应排查……""合规整改建议" |
| **客观研究** | 身份不明 / 纯学理梳理 | 仅做制度变化的客观解读，不预设委托或业务场景 | 中立陈述 |

### B. 三档解读深度（决定篇幅与工序）

| 档位 | 篇幅参考 | 包含内容 | 是否联网/查企 |
|------|----------|----------|---------------|
| **速览** | ≤1500 字 | 新旧对比表 + 3-5 条核心变化要点 | 否（仅法律检索后端取文本） |
| **标准** | 3000–5000 字 | 速览 + 核心制度深度解析 + 本行业合规清单 | SEARCH.* 取官方解读 |
| **深度** | 8000 字+ | 标准 + 域外对标 + 争议解决预判 + 结合特定公司的落地建议 | SEARCH.* + 工商信息后端（BIZ.*，见 qoder.md）（如需落地） |

### C. 应用目标适配（决定语气与导出格式）

| 应用目标 | 语气风格 | 结构侧重 | 默认导出 |
|----------|----------|----------|----------|
| **公众号/对外发布** | 传播友好、标题有钩子、克制法言法语 | 强导读、金句小标题、对比表前置 | md + 可选 HTML（内置直接生成） |
| **内部分享/法务简报** | 务实、要点先行 | 影响矩阵 + 合规清单 | md + 可选 Word（Word 转换后端, DOCX.md_to_docx, preset=legal） |
| **客户/业务部门培训** | 通俗、问答式、可照做 | Checklist 与流程卡为主 | md + 可选 Word |
| **个人研究/底稿** | 严谨完整、保留推演 | 制度解析 + 域外对标 | md |

> 视角来自**画像自动判定**；深度、应用目标、是否结合特定公司**主动询问用户一次**（见第 0 步）。

## 第 0 步：画像捕捉 + 应用参数询问（必做，先于任何检索）

1. **读取画像**：尝试读取套件根目录 `qoder.md`，提取以下字段用于定制解读落点：
   - 执业身份（律师 / 法务 / 其他）→ 决定**视角**
   - 法务场景：所在公司、所处行业、业务形态
   - 律师场景：主要服务客户所在行业、执业领域
   - 风险偏好（保守/进取）→ 影响风险分级措辞与提示密度
   - 画像缺失或字段不全 → 在下一步询问中补齐"面向行业"一项。

2. **主动询问用户一次**（用 AskUserQuestion，一个批次问完，不要逐条追问）：
   - **应用目标**：公众号/对外发布 ｜ 内部分享/法务简报 ｜ 客户/业务部门培训 ｜ 个人研究底稿
   - **解读深度**：速览 ｜ 标准 ｜ 深度
   - **是否结合特定公司落地**：否（仅做行业普适解读）｜ 是（提供公司名 → 触发工商信息后端/联网核验经营与风险，给定制化整改动作）
   - **面向行业/客户领域**：默认取画像值；画像无则让用户指定（如金融/TMT/制造/跨境贸易/医疗/房地产/能源等）

3. 记住本次选择，全程不再重复询问。纯学理梳理（用户明确只要客观制度解读、无落地场景）→ 视角强制为「客观研究」，可跳过"特定公司落地"。

## 工作原则

1. **对比表是基石**：任何档位都必须先产出新旧对比表，后续解读全部由对比表派生，杜绝脱离条文的"泛泛而谈"。
2. **每引必校验**：正文每引用一条法条/条号/生效日期，立即经法律检索后端核验原文、条号、时效与修订状态；核验不通过用权威原文替换或标注，**绝不用记忆编造法条**。
3. **实务映射而非法条摘要**：制度提炼必须答"改变了什么游戏规则 + 对本行业意味着什么 + 该做什么动作"，而非复述条文。
4. **不假设未给信息**：不替用户脑补事实；结合特定公司时所有结论须基于工商信息后端/联网取得的可核验信息，缺口明确标注。
5. **时效与冲突**：优先"现行有效"文本；标注生效日期、过渡期、新旧衔接；上位法>下位法、新法>旧法、特别法>普通法。
6. **客观中立**：解读用于识别风险与机会，负面影响与不确定性同样如实呈现，不为单方背书。
7. **积分意识**：案例/详情类调用先取摘要，确需全文再调 detail 类工具，调用前向用户说明。
8. **言简意赅，实事求是**；结论加粗，避免晦涩法言法语。

## 三步走工作流

### 阶段一 · 精准锚定与对比（基石步骤）

> 📎 加载 `references/comparison-risk-industry.md` 作为新旧对比表规范与风险分级矩阵参考。

1. **取新法全文**：用 `LAW.statute_search` 按名称/文号定位 → `LAW.statute_detail` 取正式稿全文。
2. **取对比基准（旧法/草案）**——按答复"法律检索后端优先 + 多源降级"：
   - 优先经法律检索后端检索被修订/废止的旧法或上一版本全文；
   - 法律检索后端取不到 → SEARCH.* 取征求意见稿/官方对照说明/废止令，并标注来源；
   - 仍取不到 → 在对比表中标注 `[需人工核对]`，继续完成可比部分，绝不编造旧法内容。
3. **生成新旧对比表**（Markdown，全文基石）：

   **⚠ 硬性格式要求：逐条引用原文，不得仅写"要点"概括。** 每条有变化的条款用独立小节呈现，结构如下：

   ```markdown
   ### 第X条：{条文主题}

   **新解释第X条**：

   > {引用新法该条完整原文}

   **旧解释第X条**：

   > {引用旧法该条完整原文}

   **变化**：~~改~~ / [+] 新增 / [-] 删除。{核心变化简述，包括实务影响分析}
   ```

   - 变化类型标注：新增 `[+]`；删除 `[-]`；修改用 `~~改~~`。
   - **原文引用规则**：新旧法条文必须引用法律检索后端取回的正式原文，不得缩写、概括或省略关键字句。如单条原文过长（超过 500 字），可省略与变化无关的段落并以"……（省略）"标注，但涉及变化的关键句段必须完整引用。
   - 速览档对比表即为主体；标准/深度档对比表作为后续解析的索引。

### 阶段二 · 深度解构（基于对比表，标准/深度档）

4. **提炼核心制度**：从对比表中挑出 3-5 个真正"改变游戏规则"的制度（如"安全港""分级管理由三级变五级""备案改审批"），逐个说明立法意图、与旧规差异、实务后果。
5. **风险量化分级**：对每项变化打标签——
   - 🔴 **高危**：涉刑事责任 / 高额罚款 / 业务资格丧失
   - 🟡 **中危**：行政备案/许可 / 合同效力 / 整改期限
   - 🔵 **低危**：内部流程 / 文本修订 / 审批权限调整
6. **行业影响矩阵**：针对画像/用户指定的行业，列"业务场景 → 受影响点 → 具体合规动作"（动作要可执行，如"修订委托营销协议第 X 条""新增数据出境申报流程"）。
7. **域外对标**（仅深度档）：如涉数据/反垄断/合规等有国际可比性的领域，简要对比域外制度（如 GDPR、域外 AEO）以衬托立法取向，须标注来源、不编造。

### 阶段三 · 动态校验与落地（输出前）

8. **时效性与状态校验**：经法律检索后端确认新法是否已生效、有无后续修正、过渡期安排；关键数额/期限/条号逐一回校原文。
9. **结合特定公司落地**（仅用户选"是"）：用工商信息后端核验该公司主体、经营范围、资质与现有风险，把行业影响矩阵收敛为该公司的定制整改清单（含优先级）；信息不足处明确标注，不臆测。
10. **交叉验证关键条款**：对罚款数额、生效日期等关键点，官方解读与律所/实务文章口径不一致时如实并列，不强行取一。

## 报告结构（Markdown）

{阶段一产出的对比表，全文基石。每条用独立小节，引用新旧法原文段落，标注变化类型与核心变化简述。}

## 三、核心制度深度解析  [标准+]
{阶段二提炼的 3-5 项制度，每项含：变化点 / 立法意图 / 与旧规差异 / 实务后果 / 风险等级标签}

## 四、行业影响与合规清单  [标准+]

|---------|---------|---------|----------------------|

## 五、结合本公司的落地建议  [按需·选"是"时]
{基于工商信息后端核验的该公司定制整改清单 + 优先级；信息缺口标注}

## 六、域外对标  [深度]
{与域外可比制度的简要对照，标注来源}

## 七、争议解决预判  [深度]
{新规对诉讼举证责任、管辖、裁判口径的潜在影响，引相关案例/条文}

## 八、依据与来源清单  [核心]
**新法/旧法**：{名称+文号+条号+时效}

## 输出与交付

> **⛔ 交付前必过闸门**（见 qoder.md「法律核验闸门」）：核验对象为解读全文（含法条生效日期）。

1. **核心交付为 `.md`**：命名 `新法解读_{法规简称}_{日期}.md`，保存至用户指定文件夹或当前输出目录，以 `file://` 链接交付。
2. **按应用目标可选导出**（询问或直接按第 0 步选择执行）：
   - 公众号/对外发布 → 直接生成排版友好的 HTML 文件（内置能力，无需外部 MCP）；
   - 内部分享 / 客户培训 → 调 `Word 转换后端`（DOCX.md_to_docx）（preset=legal）生成 Word。
3. **输出后简报**：说明法律检索后端调用了哪些工具各几次、检索到的新旧法版本、SEARCH.* 关键词、工商信息后端是否启用及消耗提示。
4. **md 优先**：md 完成并经 `/法律核验` 放行后，询问用户是否进一步转 Word（Word 转换后端（DOCX.md_to_docx, preset=legal））。

## MCP 降级策略

| MCP 不可用 | 降级方案 |
|-----------|----------|

## 硬性约束

- **禁止编造法条/旧法/数额/生效日期**：一切以法律检索后端原文或用户提供的正式文本为准，取不到如实标注。
- **对比表不得省略**：任何档位都必须先有新旧对比表。
- **对比表必须引用原文**：每条对比必须引用新旧法的完整原文段落，不得仅以"要点"或概括性描述代替；原文过长可省略无关段落（标"……（省略）"），但涉及变化的关键句段必须完整引用。
- **不假设未给信息**：结合公司落地的结论须基于可核验信息，缺口明确标注。
- **仅 .md 为核心交付**；HTML/Word 仅按应用目标可选导出。
- 报告末尾必须附**法律声明**免责条款。
- 第 0 步未完成画像捕捉与参数询问前，不开始检索。

## 本技能不做什么

- **不替代正式法律意见**：输出是解读底稿，最终口径由律师/法务负责。
- **不编造旧法或对比基准**：取不到对比文本时如实标注，不臆造"旧规如何如何"。
- **不脱离条文空谈**：所有解读必须能回指到对比表的具体条款。
- **不替用户脑补公司情况**：未经核验的公司信息不写入落地清单。
- **不做个案胜诉预测**：争议解决预判仅为方向性提示，个案分析交 `/要件攻防分析`、`/法律研究报告`。

## 错误处理

- 信息不足（无法名/文号/文本） → 补问，不开始检索
- 第 0 步参数未确定 → 先问应用目标/深度/落地，再推进
- 法律检索后端返回空 → 如实告知，提示用户粘贴正式文本，不编造
- 旧法取不到 → 对比表标 `[需人工核对]`，继续可比部分
- SEARCH.* 超时/不可用 → 降级为仅法条解读并声明
- 行业不明 → 第 0 步询问中要求用户指定面向行业
