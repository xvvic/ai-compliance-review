# 新旧对比表规范 · 风险分级 · 行业影响矩阵

> 本文档规范本技能阶段一（对比）与阶段二（解构）的产出标准。

## 一、新旧对比表规范

### 表格结构

| 序号 | 新法条款（要点） | 旧法/草案条款（要点） | 变化类型 | 核心变化简述 |
| :--- | :--- | :--- | :--- | :--- |

### 变化类型标注

- **新增**：标 `[+]`，新法有、旧法无的制度/义务/罚则。
- **删除**：标 `[-]`，旧法有、新法删去的内容（旧法栏写明，新法栏标删除）。
- **修改**：用 `~~旧内容~~ → 新内容` 表达条文措辞或数值变化。
- **未变但需提示**：标 `[=]`，条文未变但因体系变化而适用口径改变的，简述原因。

### 编制原则

1. **逐条对应**：以新法条文为序，逐条找旧法/草案对应条款；找不到对应即为"新增"。
2. **要点化**：栏内写要点而非全文，全文留待"核心制度解析"展开。
3. **取不到旧法时**：标 `[需人工核对]`，完成可比部分，**绝不编造旧法内容**。
4. **数值变化醒目**：罚款额、比例、期限、级别数量等数值变化必须在"核心变化简述"里点出。

## 二、风险分级标准

对每项变化打显性标签，便于读者抓重点：

| 标签 | 含义 | 典型情形 |
|------|------|----------|
| 🔴 **高危** | 涉刑事责任 / 高额罚款 / 业务资格丧失 / 不可逆后果 | 新增刑事罚则、罚款上限大幅提高、撤销许可、列入严重失信 |
| 🟡 **中危** | 行政备案或许可 / 合同效力 / 限期整改 | 新增备案义务、合同条款强制要求、设置过渡期整改 |
| 🔵 **低危** | 内部流程 / 文本修订 / 审批权限调整 | 章程或内控文件修订、内部审批层级变化 |

分级时结合用户**风险偏好**：画像为"保守"时，临界情形从高标注并加提示；不确定一律就高标注并说明理由。

## 三、行业影响矩阵编制法

把制度变化翻译为具体行业的可执行动作：

| 字段 | 要求 |
|------|------|
| 业务场景 | 用用户行业的真实业务语言（如金融"网络营销跳转"、制造"供应链尽调"、跨境贸易"AEO 认证"） |
| 受影响点 | 指明被新规触及的具体环节 |
| 风险等级 | 套用上节🔴/🟡/🔵 |
| 合规动作 | 必须可执行：如"修订委托营销协议第 X 条""新增数据出境申报流程""更新资质台账" |

### 行业术语映射（精准切入业务场景）

为避免泛泛而谈，按行业建立"法言→业务语"映射，分析时切入业务场景而非复述法条：

| 行业 | 高频业务关切 |
|------|--------------|
| 金融 | 助贷、网络营销跳转、消费者适当性、数据出境、牌照与备案 |
| TMT/互联网 | 算法推荐、个人信息处理、数据分类分级、平台责任 |
| 制造业 | 供应链尽调、产品质量与召回、出口管制、环保合规 |
| 跨境贸易 | AEO 认证、海关信用、原产地、关税与申报、失信修复 |
| 医疗医药 | 注册备案、临床数据、医疗器械分类、反商业贿赂 |
| 房地产/建筑 | 预售资金监管、招投标、建设工程合同效力 |
| 能源/环保 | 排污许可、双碳合规、安全生产、行政处罚 |

> 用户行业不在上表时，按其业务形态临时归纳关切点，不强行套用。

## 四、域外对标（仅深度档）

适用于具有国际可比性的领域（数据保护、反垄断、合规、出口管制等）。要求：
- 简要对比域外可比制度（如 GDPR、域外 AEO/竞争法），用以衬托国内立法取向；
- **标注来源、不编造**；无可靠来源则省略本节。

## 五、争议解决预判（仅深度档）

预判新规对诉讼实务的影响，可经法律检索后端案例检索佐证：
- **举证责任**：是否新增/转移举证责任、是否设推定规则；
- **管辖**：是否影响级别/地域管辖或新增专门程序；
- **裁判口径**：新规施行后裁判标准可能的变化方向（引相关条文/案例，标注为方向性预判）。
