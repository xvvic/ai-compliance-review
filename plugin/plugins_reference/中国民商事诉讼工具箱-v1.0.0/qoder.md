## 共享护栏

以下规则适用于本套件所有技能，不可覆盖：

1. **无沉默补充** -- 缺信息标标记，不编造
2. **法条核实闸门** -- 正式文件输出前必过 `/法律核验`，全部准确方可放行
3. **文档解析前置** -- 收到上传文件后先解析为 Markdown 再进入分析，绝不臆造文档内容
4. **请求权基础思维** -- 文书类技能以请求权基础方法为分析起点
5. **案件数据真实性** -- 绝不臆造 ID/记录，缺失时向用户索取
6. **保密与隔离** -- 不跨案件泄露信息；案件配置互不污染
7. **md 优先输出** -- 先 md → 核验 → 放行 → 询问转 Word
8. **案件管家自动同步** -- 任何技能产出案件相关成果后自动同步更新案件管家
9. **后端平台无关** -- 不硬编码特定MCP工具名，按能力槽（LAW/BIZ/PARSE/COLLAB/SEARCH）运行时探测使用

## MCP 预检协议

> 各技能不必重复以下协议全文，只需声明自身使用的后端类别（如 `LAW.*`），即自动继承此协议。

### 后端探测

执行任何需要外部能力的技能前，先调用 `qw_mcp_list` 探测当前会话可用的 MCP 服务：

| 后端类别 | 探测关键词 | 映射能力槽 |
|---------|-----------|-----------|
| 法律检索 | `law` / `yuandian` / `pkulaw` / `法宝` | LAW.* |
| 工商信息 | `qcc` / `企查查` / `天眼查` / `biz` | BIZ.* |
| 协作平台 | `dws` / `dingtalk` / `钉钉` / `lark` / `飞书` | COLLAB.* |
| 全网搜索 | `全网搜索` / `web_search` / `search` | SEARCH.* |
| 文档解析 | `parse` / `mineru` / `百炼` / `markdown` / `docling` | PARSE.* |
| Word 转换 | `md_to_docx` / `docx` / `word` | DOCX.md_to_docx |
| 知识库 | `qmind` / `knowledge` / `notebook` | qmind |

### 探测结果处理

1. **全部可用** → 静默进入工作流。
2. **部分不可用** → 用 `AskUserQuestion` 告知用户：

   > 检测到 {不可用后端名} 尚未连接。{该后端对本技能的具体作用}。
   > 建议前往 **QoderWork → 设置 → 连接器** 中搜索配置。
   > 是否暂不配置、以降级模式继续？

   - 用户选择配置 → 等待配置完成后重新探测
   - 用户选择继续 → 降级执行，对应位置标注 L4 标签

3. **全部不可用** → 同上询问后降级执行。

### 降级规则

| 后端类别 | 降级标注 | 降级行为 |
|---------|---------|---------|
| 法律检索 (LAW.*) | `[L4-法条待验证]` | 保留法条引用但标注，继续生成 |
| 工商信息 (BIZ.*) | `[L4-主体信息待验证]` | 保留用户提供的主体信息但标注 |
| 协作平台 (COLLAB.*) | `[本地模式-协作平台未连接]` | 回退本地 JSON + Markdown |
| 全网搜索 (SEARCH.*) | `[L4-信息待验证]` | 依赖模型内置知识并标注 |
| 文档解析 (PARSE.*) | `[视觉降级-待人工复核]` | 按降级链回退：内置 pypdf → fitz 视觉降级 → 内置 Read；全部失败则停下报告 |
| Word 转换 (DOCX.md_to_docx) | 无标注 | 仅交付 Markdown 并提示用户在本机手动转换 |
| 知识库 (qmind) | `[知识库不可用]` | 仅依赖对话内上下文，不持久化知识 |

**核心原则**：后端不可用 **不阻塞** 文书生成，仅在对应位置加注标签。

**MCP 配置引导**：降级模式下，在输出文件顶部附加：`"当前以降级模式运行。如需启用自动校验，请前往 QoderWork → 设置 → 连接器 配置对应 MCP 服务。"`

## 法律核验闸门

> 各技能不必重复闸门全文，只需写一行 `> ⛔ 交付前必过闸门（见 qoder.md）`，并注明核验对象（如"诉状全文"）。

交付正式文件之前，调用 `/法律核验` 对文书全文的法规、法条、案例引用逐条核验：

- **放行标准**：判定全为「准确 / 准确(有省略)」
- **失败处理**：出现「时效风险 / 语义不一致 / 未命中」→ 修正后重新核验
- **降级处理**：法律检索后端不可用时按 MCP 预检协议降级标注 `[L4-法条待验证]` 并提示人工复核
- **底线**：不得跳过闸门

> 各文书技能不必重复排版表，只需注明使用哪套 preset（正式文书 / 工作底稿）。

### 输出流程（md 优先）

核验放行后先产出 md 成品（写入输出目录并回链），随后询问用户「是否转 Word（`DOCX.md_to_docx, preset=legal`）」，确认才转 .docx。
