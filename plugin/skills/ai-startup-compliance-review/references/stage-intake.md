# Stage: Intake

Use this file when facts are missing or the scenario is ambiguous.

## Intake Principles

- Ask only questions that change risk classification or review path.
- Keep the first intake to five questions or fewer unless the user asks for a full interview.
- Offer assumptions for fast review when the user cannot answer immediately.
- Capture facts as `[用户材料]` and assumptions as `[模型推理-待核验]`.

## Core Questions

1. What is being reviewed: financing, data use, launch, cross-border deployment, or vendor contract?
2. What AI system is involved: model type, function, users, deployment stage, and whether outputs affect rights, prices, eligibility, or safety?
3. What data is used: personal information, sensitive personal information, minors, public web data, customer documents, confidential data, important data, or overseas data?
4. Who are the parties and roles: provider, deployer, vendor, processor, controller, customer, investor, or end user?
5. What decision is needed: proceed, proceed with conditions, pause, stop, or prepare a diligence/report artifact?

## Scenario-Specific Follow-Ups

- Financing: ask for cap table/diligence context, core IP ownership, training data provenance, open-source dependencies, outstanding disputes, and investor representations.
- Data processing: ask lawful basis, notice/consent, retention, sharing, vendor processing, export, and deletion rights.
- Product launch: ask release date, user base, AI disclosure, testing results, complaint path, logging, marketing claims, and rollback plan.
- Cross-border: ask target jurisdictions, data transfer paths, hosting location, local users, provider/deployer role, and sanctions/export-control exposure.
- Vendor contract: ask whether vendor trains on prompts/uploads, data residency, deletion, subprocessor list, incident notice, audit rights, output/IP terms, and liability cap.
