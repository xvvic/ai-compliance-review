## 我们最终要做的系统是什么

一个AI公司合规审查工作流系统（Compliance Workflow System）：

- 输入业务场景（融资 / 数据处理 / 上线 / 出海 / 合同）
- 自动识别合规风险
- 进行风险分级（低/中/高）
- 调用法律知识库进行检索
- 输出合规审查路径 + 建议 +依据

## 二、核心技术形态

系统大致会涉及：

- 合规规则库（结构化知识）
- workflow / agent 分级决策链
- 风险分类引擎（rule + LLM）

## 三、参考系统

我们会参考并可能基于类似能力设计：

- 北大法宝 MCP（法律检索与合规数据能力）https://mcp.pkulaw.com/
    这个平台提供：法律法规语义检索；法条查询接口（search_article / get_article）；合规相关知识调用能力；可作为法律知识库底座参考
    
## 四、技术同学可以重点关注的结构

建议重点理解以下模块：

1. 输入层：案情 / 合同 / 数据 / 业务描述
2. 检索层（核心）：法条检索（MCP / embedding search）/案例检索/类似制度匹配
3. 决策层：风险分类（L1/L2/L3）rule-based + LLM hybrid
4. 工作流编排：step-by-step合规审查链路/类 agent flow
5. 输出层：合规报告生成风险提示 + 修改建议
