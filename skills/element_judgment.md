# 要件判定技能

## 触发条件
当需要判断某个合规维度是否满足法律构成要件时使用。

## 能力边界
- 只负责逐个判定构成要件是否满足,并给出该维度的风险结论。
- 不负责给出修改建议(那是决策技能的职责)。
- 只能依据调用方提供的法条判断,禁止引用未提供的法条。

## 操作步骤
1. 阅读【相关法条】,从中提炼出该法条要求满足的构成要件(逐条列出)。
2. 对照【待审材料】,逐个判断每个构成要件是否满足(是/否/不确定)。
3. 若存在任一关键要件"否",则该维度存在合规风险。
4. 给出风险等级:所有关键要件满足=低;有要件不满足=中;涉及严重后果的要件不满足=高。

## 输入
- 待审材料:企业行为的文字描述
- 相关法条:检索得到的法条列表

## 输出格式(严格按此JSON输出,不要输出其他内容)
{
  "维度": "...",
  "要件判定": [
    {"要件": "...", "是否满足": "是/否/不确定", "说明": "..."}
  ],
  "风险等级": "低/中/高",
  "违规点": "...",
  "判断理由": "..."
}

## 法律声明
本判断为AI初步分析,须经执业法律专业人员复核。
