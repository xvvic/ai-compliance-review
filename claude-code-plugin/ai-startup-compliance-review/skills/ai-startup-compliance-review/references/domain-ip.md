# Domain: Intellectual Property

Use this file for training data copyright, generated outputs, model ownership, open-source software/model licenses, employee/contractor ownership, and trade secrets.

## Review Areas

- Training data rights: copyright, database rights, website terms, licenses, consent, confidentiality, and provenance.
- Model ownership: base model license, fine-tuning rights, weights, prompts, embeddings, RAG corpus, and evaluation data.
- Generated output: ownership, usage rights, infringement risk, similarity risk, and vendor restrictions.
- Software and open-source: code licenses, model licenses, attribution, commercial-use limits, copyleft, redistribution, and acceptable use.
- People and vendors: employee invention assignment, contractor work-for-hire/assignment, university or prior employer claims.
- Trade secrets: access control, confidentiality, logging, clean-room procedures, and disclosure handling.

## Red Flags

- Core training data lacks source, license, or collection record.
- Customer confidential data used for model improvement without contract permission.
- Open-source model or code has commercial restrictions inconsistent with product plan.
- No assignment from key contractor, founder, intern, or research collaborator.
- Generated outputs are marketed as exclusive or infringement-free without evidence.
- Investor materials call the model "proprietary" while core functionality depends on third-party APIs.

## Remediation Artifacts

- Dataset provenance register.
- OSS/model bill of materials.
- License review memo.
- Invention assignment packet.
- Contractor assignment confirmation.
- Trade secret access matrix.
- Output-risk disclaimer and review workflow.
