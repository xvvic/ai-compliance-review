# Stage: Remediation

Use this file when converting risks into action plans.

## Remediation Format

Every remediation item needs:

- Action.
- Owner.
- Evidence artifact.
- Acceptance criterion.
- Deadline or review gate.
- Residual risk after completion.

## Action Patterns

| Risk type | Practical remediation |
|---|---|
| Missing lawful basis | Map data flow, identify lawful basis, revise notice/consent, stop unsupported processing |
| Training data provenance gap | Build dataset register, classify source rights, remove unsupported sources, retain license evidence |
| Sensitive/minors data | Add explicit necessity review, enhanced consent, minimization, access controls, deletion and audit |
| Vendor training on customer data | Negotiate no-training clause, confidentiality, deletion, subprocessor controls, incident notice |
| AI safety/hallucination | Add evals, human review, output restrictions, logs, incident process, user disclosures |
| IP ownership ambiguity | Collect assignments, contractor agreements, OSS/model license review, invention records |
| Marketing claim risk | Substantiate claims, narrow language, add limitations, legal review before publication |
| Cross-border data risk | Map transfer path, assess mechanism, localize where necessary, add transfer impact review |

## Acceptance Criteria Examples

- Dataset register covers source, license, collection method, PI status, retention, and exclusion decision.
- Product launch checklist shows completed safety tests, rollback owner, complaint path, and user disclosure.
- Vendor contract includes no-training/no-retention terms or written exception approved by counsel.
- Investor disclosure schedule lists unresolved data/IP/regulatory matters with mitigation status.
- Data export review records transfer purpose, recipient, data categories, mechanism, and residual risk.
